Локации (админка): /admin/locations

Файл: /docs/documentation/admin-locations.html · привязка к группам: location_team · переиспользуемые UI: partial'ы
Справочник филиалов/залов партнёра. Все маршруты раздела в routes/web.php обёрнуты в middleware('can:locations.view'). Без locations.view — 403 на index, data, show, store, update, delete.
Создание, изменение и удаление записей — дополнительно locations.manage (middleware на POST/PUT/DELETE). При только locations.view страница и чтение доступны, модалки create/edit и кнопка «Добавить» скрыты.

2) Интерфейс страницы

View: resources/views/admin/locations/index.blade.php. Общие блоки описаны в Переиспользование UI-блоков.

  • Toolbar — карточка payments-report-surface, кнопки «Добавить» (locations.manage), «Фильтры», «Колонки»; CSS admin-list-toolbar.css (как у /admin/users и отчётов).
  • Фильтры (collapse): name (название/адрес), status — active/inactive; плюс глобальный поиск DataTables.
  • DataTables — server-side, pageLength: 10, русификация: @include('partials.datatables.ru').
  • Колонка «Группы» — только если у партнёра есть хотя бы одна группа ($teamOptions не пуст). В ячейке краткая подпись; полный список — в tooltip через @include('partials.ui.hover-list-dropdown') и KidsCrmHoverListDropdown.renderCell(teams_label, teams_titles); init на событии draw.dt.
  • Модалки create/edit — при locations.manage и непустом $teamOptions: multiselect team_ids[] на Select2 (@include('partials.select2.teams-multiselect'), поля #locationCreateTeamIds, #locationEditTeamIds). Ошибки валидации team_ids подсвечиваются через KidsCrmTeamsMultiselectSelect2.markInvalid.
  • Удаление — кнопка #locationDeleteBtn в edit-модалке, подтверждение через общий showConfirmDeleteModal.

3) Привязка групп (team_ids)

В модалках create/edit (только при locations.manage и наличии групп у партнёра) — multiselect team_ids[] (все группы партнёра из Team::where(partner_id), сортировка по title).

  • Создание/обновление: LocationTeamSyncService::syncTeamsForLocation — полная замена списка.
  • Валидация team_ids.* — только id групп текущего партнёра (StoreLocationRequest / UpdateLocationRequest); чужой id → 422 «Выберите группу из списка текущего партнёра».
  • GET …/locations/{id} возвращает team_ids: number[] при locations.view.
  • В контроллере есть дополнительная проверка: без locations.view sync team_ids не выполняется (на практике store/update недоступны без view из‑за middleware группы маршрутов).

Обратная сторона (привязка локаций к группе) — /admin/teams, поле location_ids.

4) Таблица и /admin/locations/data

  • Фильтры запроса: name, status (active/inactive), поиск DataTables (search.value); при одновременной передаче приоритет у панельного name.
  • При locations.view в каждой строке JSON:
    • teams_label — краткая подпись для таблицы;
    • teams_label_full — полный список названий через запятую;
    • teams_titles — массив названий (отсортирован по title группы) для tooltip.
    Правила teams_label: 0 групп — пусто; 1–2 — все названия; 3+ — «Первая группа, еще N шт.» (N = всего − 1).
  • Без групп у локации: teams_label / teams_label_full — пустые строки, teams_titles — [].
  • Колонки в «Поля списка»: id, name, address, teams_label (если есть группы у партнёра), is_enabled_label, actions (при locations.manage).

5) Связанные сущности (не путать)

  • school_leads.location_id — локация заявки, не ученика.
  • team_schedule_slots.location_id — локация слота расписания.
  • payments.location_id — снимок в журнале оплат.
