Расписание школы (календарь абонементов)

Страница входит в раздел «Абонементы» как вкладка «Расписание школы». Доступ к экрану и JSON календаря задаётся правом lessonPackages.view (группа маршрутов в routes/web.php с middleware can:lessonPackages.view). Отдельно: управление слотами расписания групп (scheduleSlots.manage / просмотр) описано в коде прав и тестах календаря; здесь — клиентская часть календаря и модалки слота.

Данные недели для сетки

Фронт подгружает слоты через GET /admin/lesson-packages/school-schedule/week (параметры week, опционально location_id). Каждый элемент массива occurrences содержит, среди прочего, поле team_title — название группы (команды) для отображения в ячейке и в UI.

Фильтр «Локация» в тулбаре

Блок с выбором локации (id="schoolCalLocation") рендерится только при наличии права locations.view (директива Blade @can('locations.view')). У базовой роли admin в конфиге config/role_base_permissions.php это право включено; у кастомных ролей его можно отозвать — тогда фильтр не показывается, но календарь по-прежнему доступен при lessonPackages.view (все слоты партнёра без фильтра по локации).

Модалка сведений о слоте

Контейнер: #schoolCalSlotModal. При открытии слота данные подставляются из выбранного occurrence на клиенте.

Элемент Поведение
#schoolCalSlotModalTitle Заголовок модалки — название группы (team_title из occurrence). Статического текста «Слот расписания» в заголовке нет.
#schoolCalSlotSummaryWhen Строка даты и времени слота; оформление через класс school-cal-slot-summary__when (компактный размер шрифта в стилях вкладки).
Строка «Группа» в теле модалки Не выводится — название группы дублируется только в заголовке модалки.
#schoolCalSlotSummaryLoc Строка «Локация» / значение локации показываются только при locations.view. Без права разметка строки не попадает в HTML.

Привязка абонементов и лимит строк календаря

При успешной привязке гибкого, фиксированного или разового абонемента к слотам создаются строки user_team_schedule_slots с заполненным user_lesson_package_id. Поле user_lesson_packages.lessons_remaining при этом не изменяется — оно корректируется только при смене статуса занятия (см. ниже). Ограничение по объёму абонемента выражено в том, что число таких строк по назначению не может превысить lessons_total. Детали периода starts_at/ends_at и логики фиксированной цепочки — в разделе «Абонементы».

Статусы занятия и списание остатков

Для каждой записи ученика на слот и дату (абонемент или пробное) в календаре школы можно сохранять статус из справочника партнёра lesson_occurrence_statuses. У статуса есть флаг consumes_lesson (например, «Посетил» и «Не посетил» — списывают, «Запись», «Отмена», «Заморозка» — нет; точный набор создаётся сидером и может быть расширен в админке с учётом этого флага).

  • Абонемент: дельта к user_lesson_packages.lessons_remaining считается по переходу «предыдущий статус списывал / новый списывает» (UserLessonPackageConsumptionAdjuster). Повторное сохранение того же типа списания не меняет остаток дважды.
  • Пробное: для строки с is_trial_lesson корректируются trial_lessons_remaining / trial_lessons_total (SchoolScheduleTrialLessonConsumptionAdjuster).

POST сохранения: /admin/lesson-packages/school-schedule/occurrence-status (имя маршрута admin.lesson-packages.school-schedule.occurrence-status.store). В ответе при успехе могут быть объекты user_lesson_package { id, lessons_remaining, lessons_total } или trial_registration { user_team_schedule_slot_id, lessons_remaining, lessons_total } — их использует фронт для обновления счётчиков в модалке.

GET истории по паре слот + дата + ученик (+ опционально user_lesson_package_id для различения записей): /admin/lesson-packages/school-schedule/occurrence-status/history.

JSON недели: регистрации в ячейке

Каждый элемент occurrences[] содержит массив registrations[]. Для строк из БД формируются поля (среди прочего):

  • registration_kind — trial или package;
  • lessons_remaining, lessons_total — для абонемента берутся из связанного user_lesson_packages, для пробного — из trial_lessons_* строки календаря;
  • line, user_label, current_status, occurrence_status_history_count — для отображения в модалке «Записаны на занятие».

Сервис форматирования: TeamScheduleCalendarService::formatSlotRegistrationLine.

Пробные занятия: добавление, баланс и ограничения

Пробное — запись без абонемента: user_team_schedule_slots с is_trial_lesson = true, user_lesson_package_id = NULL. Пробное действует только в выбранную дату: ends_at равно starts_at. При создании через API задаются trial_lessons_remaining = 1, trial_lessons_total = 1. Списание единицы пробного «урока» происходит при выборе статуса с consumes_lesson = true (аналогично абонементу, но поля другие).

У пользователя в таблице users хранится флаг has_used_school_schedule_trial: после успешного создания пробной записи он выставляется в true. Пока существует строка пробного у ученика (is_trial_lesson = true и user_lesson_package_id IS NULL), вторую создать нельзя; те же условия проверяются в SchoolCalendarTrialEligibilityService и при создании (блокировка строки в транзакции). Ячейка слота/даты не должна быть занята абонементом или посторонней записью.

GET …/trial-registration-eligibility — заранее проверяет возможность добавления (ответ allowed, reason).

UI: «(Отменить)» в списке записей

В модалке слота в блоке «Записаны на занятие» для пробной записи рядом с подписью «Пробное занятие» выводится ссылка (Отменить). Ссылка доступна пользователям с правом lessonPackages.view (так же, как и сам экран) и снимает пробную запись с занятия.

API: удаление (отмена) пробного занятия

Для отмены используется эндпоинт:

  • DELETE /admin/lesson-packages/school-schedule/trial-registration/{userTeamScheduleSlot}

Идентификатор берётся из JSON недели (каждая регистрация содержит поле user_team_schedule_slot_id).

Побочные эффекты отмены

  • Очистка статусов: удаляются все записи в user_lesson_occurrence_status_events для этой даты/слота/ученика, относящиеся к пробному (условие user_lesson_package_id IS NULL).
  • Аудит: создаётся запись в my_logs с type = 60 («Расписание») и action = 601 («Отмена пробного занятия в расписании»), с заполненными target_type/target_id/target_label.

Связь с логикой абонементов

Период назначения starts_at/ends_at, лимит строк календаря по lessons_total и отсутствие списания lessons_remaining при самой привязке описаны в «Абонементы: назначение и период с календаря школы». Текущая страница дополняет это правилами статусов, пробного баланса и полей JSON недели; разметка модалки и право locations.view — только отображение.
