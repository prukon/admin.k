Локации ↔ группы (location_team)

Файл: /docs/documentation/location-team-bindings.html · миграция pivot: 2026_05_28_100000_create_location_team_table.php
Суть: справочник локаций (/admin/locations) и групп (/admin/teams) связываются many-to-many через таблицу location_team. У ученика нет поля users.location_id (удалено миграцией 2026_05_28_110000_drop_location_id_from_users_table) — «география» ученика в отчётах и фильтрах идёт через users.team_id и привязки группы к локациям. Локация занятия в календаре — отдельно: team_schedule_slots.location_id.

1) Правила доступности группы в локации

Состояние группыДоступна в локации L
Нет ни одной строки в location_team для этой группы Да, во всех локациях партнёра («универсальная» группа)
Есть привязки только к L1, L2, … Только в перечисленных локациях
Ученик без team_id В отчёте «Задолженности» при фильтре по конкретной локации не попадает (кроме режима «Без локации»)

Реализация на сервере: App\Services\TeamLocationAvailabilityService (isTeamAllowedAtLocation, scopeAvailableForLocation, applyDebtUserTeamLocationFilter).

2) Таблица location_team

  • Колонки: partner_id, location_id, team_id, timestamps.
  • Уникальность: пара (location_id, team_id).
  • FK на partners, locations, teams — cascadeOnDelete (при hard delete родителя pivot удаляется; soft delete группы pivot сохраняется).
  • Синхронизация: App\Services\LocationTeamSyncService — полная замена списка id с обеих сторон.

3) UI и API: группы (/admin/teams)

Доступ к разделу — groups.view. Редактирование привязок к локациям — дополнительно locations.view.

  • Модалки create/edit: чекбоксы location_ids[] (активные локации партнёра).
  • POST /admin/teams, PATCH /admin/team/{id} — при locations.view ключ location_ids уходит в sync; без права ключ отбрасывается в TeamController.
  • GET /admin/team/{id}/edit — JSON location_ids (только при locations.view).
  • Фильтр таблицы: параметр location_id — id локации или none (только группы без привязок к локациям). Без locations.view параметр в запросе игнорируется.
  • Колонка «Локации» (locations_label) в data и сортировка по locations_label — при locations.view.

Подробнее: Группы (админка) — § про locations.view. Текстовые поля «Тренировочная база» / «Адрес» группы (teams.training_base, teams.address) — отдельные права, см. admin-teams §5.2.

4) UI и API: локации (/admin/locations)

Доступ — locations.view; создание/изменение/удаление справочника — locations.manage.

  • Модалки create/edit: Select2 multiselect team_ids[] (группы партнёра), partial partials.select2.teams-multiselect — только при locations.manage и наличии групп.
  • POST /admin/locations, PUT /admin/locations/{location} — sync team_ids при locations.view.
  • GET /admin/locations/{location} — JSON с team_ids.
  • В GET /admin/locations/data при locations.view — поля teams_label, teams_label_full, teams_titles (tooltip через partials.ui.hover-list-dropdown).

Подробнее: Локации (админка), UI-блоки: переиспользуемые partial'ы.

5) Расписание школы и слоты

  • Календарь: фильтр location_id в GET …/school-schedule/week — см. Расписание школы.
  • Модалки слота: в селекте группы опции с data-location-ids; при выборе локации клиент скрывает недоступные группы (filterTeamSelectForLocation в slotModals.blade.php).
  • Сервер: POST/PUT admin/team-schedule-slots — TeamScheduleSlotController + assertTeamAllowedAtLocation → 422 на team_id, если группа не подходит к location_id слота.

6) Пользователи — без локации в карточке

В разделе /admin/users нет поля, фильтра и колонки «Локация ученика». Входящий location_id в POST/PATCH /admin/users не сохраняется (колонки в БД нет). При создании клиента из заявки с сайта в ученика не копируется school_leads.location_id — локация остаётся только у заявки.

См. Пользователи §3.

7) Отчёты и платежи

ОтчётИсточник локации для фильтра
Платежи, LTV, платежи по месяцам payments.location_id — снимок при первом создании строки платежа, только если location_id явно передан в атрибуты PaymentLedgerRecorder (webhook T‑Bank сам локацию не ставит)
Задолженности Группа ученика: users.team_id + location_team. filter_location_id=none — team_id IS NULL или универсальная группа

Единый параметр UI: filter_location_id (пусто / none / id). Право: locations.view. См. Отчёты §2.1.
