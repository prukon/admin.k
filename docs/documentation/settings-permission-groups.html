Матрица прав: группы permissions (permission_groups)

Файл: /docs/documentation/settings-permission-groups.html · см. также кастомные роли, базовые права партнёра
Назначение. На вкладке «Настройки → Права и роли» права отображаются в аккордеоне по группам (permission_groups). Группа — только навигация в UI матрицы: имена прав, назначение в permission_role и проверки Gate / hasPermission() не меняются при перегруппировке.

1) Где в интерфейсе

  • Страница: GET /admin/settings/rules (вкладка «Права и роли», право settings.roles.view).
  • Разметка аккордеона: /resources/views/admin/setting/rule.blade.php — блок #permission-accordion.
  • Контроллер собирает группы: RuleController::showRules — коллекция $groups с вложенными permissions.

2) Модель данных

  • permission_groups — справочник групп (slug, name, sort_order, is_visible).
  • permissions.permission_group_id — FK на группу. Уникальность прав — по полю permissions.name.
  • Назначение прав ролям — как и раньше, только через permission_role (partner_id, role_id, permission_id).

3) Список групп (после перегруппировки 2026-06)

Раньше почти все пункты меню и полевые права сидели в одной группе «Главное меню». Сейчас каталог разбит по смысловым блокам.

4) Видимость в матрице (is_visible)

  • У каждого permission есть permissions.is_visible. Для роли не superadmin на странице показываются только права с is_visible = 1 (RuleController::showRules).
  • Группа аккордеона отображается, только если после фильтра в ней осталось хотя бы одно право. Поэтому администратор партнёра может видеть меньше блоков, чем superadmin (например, «Заявки и лиды» и «Партнёры и сервис» скрыты, если все права группы с is_visible = 0).
  • superadmin видит все права и все группы каталога (включая «невидимые» бейджи в UI).

5) Источник истины и деплой

  • Группы: /database/seeders/PermissionGroupsSeeder.php (upsert по slug).
  • Каталог прав и group_slug: /database/seeders/PermissionSeeder.php (upsert по name, обновляет permission_group_id).
  • Миграция для уже существующих БД: /database/migrations/2026_06_20_140000_reorganize_permission_groups.php (создаёт группы и переназначает permission_group_id batch-update; down() намеренно пустой).
  • Миграция страничных прав в «Главное меню»: /database/migrations/2026_06_23_120000_move_page_view_permissions_to_main_menu.php (reports.view, schedule.view, groups.view, setPrices.view, contracts.view, partner.view, users.view, settings.view + sort_order по боковому меню).
  • Миграция directories.view и пунктов «Расписание школы» / «Лиды» в mainMenu: /database/migrations/2026_06_23_140000_add_directories_view_and_menu_page_permissions.php.
  • На проде после выката: php artisan migrate. Отдельный запуск PermissionSeeder не обязателен, если миграция применена.
  • Порядок в тестах и dev: PermissionGroupsSeeder → PermissionSeeder (см. DatabaseSeeder, CrmTestCase).

6) Что не менялось

  • Строки permissions.name и логика Gate в AuthServiceProvider.
  • Базовые наборы в /config/role_base_permissions.php и автопроставление при создании партнёра.
  • Проверки в Request / Blade по тем же именам прав.
