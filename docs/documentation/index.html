Документация

Папка: /docs/documentation. В кабинете со правом «документация»: список разделов также открывается по /doc (редирект).
Это “главная” страница документации (список разделов). Файлы лежат рядом в этой же папке.

Разделы

  • Оплаты: таблицы payment_intents / payables / payments / users_prices
  • Как работать с partner_id: обычный админ и страницы superadmin (руководство для рефакторинга)
  • Партнёр‑контекст и SetPartner: current_partner, anti‑leak, блокировки
  • Партнёры: базовые роли и права по умолчанию (user/admin в разрезе партнёра)
  • Настройки: кастомные роли и стартовый набор прав (admin из конфига, UI без перезагрузки)
  • Аудит CRM (my_logs): AuditEvent, AuditLogger, колонки event/level
  • Настройки → Логи: вкладка журнала, фильтры, доступ, SUPERADMIN_ALL_OR_FILTER
  • T‑Bank (мультирасчёты): настройки, комиссии, workflow
  • T‑Bank: админка выплат (список, DataTables, карточка, изоляция по партнёру)
  • T‑Bank: возврат в отчёте «Платежи» и отмена отложенной выплаты
  • T‑Bank: СБП (QR) в интерфейсе — GetQr/Data, страница /tinkoff/qr/…, доступы
  • Очереди в админке: мониторинг, доступы, queue.log, restart worker
  • T‑Bank: глобальные настройки выплат, автовыплаты, защита от дублей, где смотреть статистику
  • T‑Bank (мультирасчёты): тесты (security/webhooks/payouts/sm-register/commissions/refunds)
  • Отчёт “Платежи” (админка): колонка и фильтр объекта, колонки, права, история T‑Bank
  • Отчёты (админка): KidsCrmDataTable, columns-settings, payment-intents / payments (0 custom), LTV/monthly (chevron custom), фильтры, AJAX/суммы
  • Абонементы: период назначения, привязка к календарю, разовое (no_schedule), лимит строк, остаток (lessons_remaining) и статусы
  • Расписание школы: календарь, фильтр «Объект» (Все / Без объекта / id), модалка слота, статусы, пробное, разовое, JSON/API
  • Журнал расписания: /schedule, статусы (statuses), тренер при «Посетил» (schedule_users.trainer_profile_id)
  • Нагрузка тренеров: вкладка на /schedule, матрица Пн–Вс, быстрый выбор месяца (3 ссылки), AJAX schedule.trainer-workload.data
  • ЗП тренеров: черновик за месяц, формула итого, autosave, слепки vN / batch, права schedule.trainerSalary.*
  • Листы ЗП: архив слепков (readonly), фильтр «Только актуальные», batch и snapshot
  • Объекты ↔ группы (teams.location_id): одна группа — один объект, sync с /admin/locations, лендинг, слоты, отчёты; удалены pivot location_team, teams.address, teams.training_base
  • Справочники: иерархия Район → Объект → Группа — обзор, вкладки, DirectoriesMenu (подпись сайдбара), права, БД, лендинг, CRM заявки, автотесты
  • Районы (админка): /admin/districts — districts.view, CRUD, multiselect объектов в edit (location_ids), колонка «Объекты» с tooltip, hard delete
  • Объекты (админка): /admin/locations — locations.view / locations.manage, поля «Район» и «Администратор» (admin_user_id), привязка групп (team_ids)
  • Виды спорта (админка): /admin/sport-types — sport_types.view / sport_types.manage, вкладка «Справочники» (не отдельный пункт меню), привязка к группам (teams.sport_type_id), фильтр на лендинге заявки
  • Группы (админка): /admin/teams — groups.view, фильтры по объекту / району / администратору объекта, «Стоимость», тренер, расписание, виды спорта
  • Тренеры (админка): /admin/trainers, роль trainer, trainer_profiles, привязка к группам, фильтр в отчётах
  • Родители и семейный кабинет: справочник parents, братья с общим parent_id, переключатель в сайдбаре, active_student, права и изоляция партнёра
  • Договоры (клиентские): список, модалка «Создать договор» (PDF и режим «форма клиенту»), карточка, подписант CRM, revoke/refund, вкладка «Шаблоны»
  • Шаблоны договоров DOCX: модалки create/edit/email, fields_schema, fill_sort_order, версии, email-приглашение
  • Заполнение договора родителем: /account-settings/documents, модалка fill, generate PDF, подпись SMS, sync профиля
  • Пользователи в админке: /admin/users, родители, колонка и фильтр «Договор» (contracts.view, сортировка по статусу), мед./особенности, группа (без users.location_id), доп. поля
  • Заявки с сайта: виджет iframe, CRM (лид → клиент → договор), toolbar, фильтры, Telegram (≠ B2B partner_leads)
  • Страница заявки партнёра: /lead/{landingSlug}, каскад район → объект → услуга, district_id, source=landing
  • Блог: публичные страницы, админка (blog.view), ИИ статей, автопубликация в VK (kidscrm): ИИ-анонс, обложка, очередь default
  • Требования к единообразию Feature‑тестов (партнёр, авторизация, права)
