Dev-фикстуры (2026‑07)

Локальная и test-БД: при SEED_DEV_DATA=true в .env блок Dev*-сидеров в DatabaseSeeder создаёт детерминированные партнёры 1–3, юр. лица (SHOP-DEV-*), справочники районов/видов спорта, группы с legal_entity_id, правила комиссий T‑Bank, историю платежей/выплат, Robokassa (1–2) и глобальный T‑Bank. На production флаг выключен. Подробно — dev-seed-data.

Изменения в справочниках (юр. лица и группы, 2026‑06)

Юр. лица (/admin/legal-entities): публичное наименование в списке и привязках — PartnerLegalEntity::displayTitle() (organization_name → fallback title); поле «Наименование организации» обязательно в модалках CRM; колонка «Регистрация в банке» — бейдж + иконка i с KidsCrmTooltip; ошибка удаления при привязанных группах — модалка showErrorModal (не alert); после удаления модалка редактирования не открывается повторно. Подробно — admin-legal-entities.

Группы (/admin/teams): при ≥2 активных юр. лицах поле «Юр. лицо» в модалках обязательно; без привязки группы T‑Bank недоступен (жёсткий guard). Колонка «Юр. лицо» — displayTitle() явной привязки; без привязки — «—». Подробно — admin-teams §5.2, payments §2.3.5.

Общее: несколько юр. лиц на партнёра; CRUD в модалках; sm-register на карточке /admin/legal-entities/{id} (legal_entities.sm_register); LegalEntityResolver для оплат — ShopCode и реквизиты только из partner_legal_entities (без fallback на partners). При одном активном юр. лице группа может быть без legal_entity_id — резолвер использует единственное юр. лицо.

Организация партнёра (2026‑06‑30): реквизиты и НДС — только в «Юр. лица»; с карточки партнёра и вкладки «Организация» в ЛК убраны legacy-поля, а также город, индекс и данные руководителя (поле title переименовано в UI в «Название школы/секции»). Legacy-колонки partners.* в БД сохранены, приложение их не пишет из форм (PartnerLegacyLegalFields). Подробно — account-partner-organization.

Изменения в оплатах (T‑Bank, 2026‑06)

Глобальный терминал T‑Bank: ключи eacq/e2c — одна запись payment_systems(partner_id=NULL) на /admin/settings/paymentSystem. Автовыплата и задержка — в правилах комиссий (tbank §5), не в платёжных системах и не в глобальной задержке settings. Per-partner payment_systems(tbank) устарели.

Юр. лицо в админке T‑Bank (снимок legal_entity_id): колонка «Организация» в списке выплат (tbank-admin-payouts §4.1); строка «Организация» и timeline из 4 шагов на карточке платежа (tbank §5.7). Колонка «Партнёр» — по-прежнему название клуба (partners.title), не юр. лицо.

Группа платежа (team_id): для всех типов оплат (месячный взнос, клубный, доп. платёж, абонемент) юр. лицо и выплата определяются через группу в payables.meta.team_id — PayableTeamResolver (payments §2.3). В админке: колонка «Группа» в доп. платежах и назначениях абонементов; backfill — миграции 2026_06_29_150000 + 2026_06_29_160000.

CloudKassir (чеки): ИНН принципала, наименование и НДС — из юр. лица по той же цепочке (forFiscalReceipt → payable → группа). Без meta.team_id и привязки группы к юр. лицу чек не собирается. СНО — из env платформы; partners.email, phone, website — по-прежнему с партнёра. Подробно — admin-legal-entities §5.1.

Feature-тесты оплат/T‑Bank/чеков: в фикстурах нельзя полагаться на legacy partners.tinkoff_partner_id / tax_id — только справочник юр. лиц + цепочка через группу. Хелперы CrmTestCase — tests-standards §5.

Multi-entity guard (2026‑06‑30): при ≥2 активных юр. лицах Init T‑Bank запрещён, если у выбранной группы нет активной привязки teams.legal_entity_id с ShopCode; ошибка Init — «Для выбранной группы не настроено юр. лицо». Клубный взнос: кнопки T‑Bank скрываются JS до выбора группы с доступным ShopCode. Подробно — payments §2.3.5, admin-legal-entities §1.1.

«Поставщик услуг» на витрине (2026‑07): на POST /payment, /payment/club-fee и публичной странице оплаты абонемента GET /pay/ulp/{token} (при T‑Bank СБП) показывается строка с юр. лицом, которому уйдёт net-выплата и которое попадёт в чек CloudKassir — та же цепочка, что при Init (PayableTeamResolver → LegalEntityResolver). Формат: {organization_name}, ИНН {tax_id}, {city}. На витрине ученика без валидного юр. лица — «Обратитесь в школу», кнопки T‑Bank скрыты; на публичной ссылке — «Обратитесь в школу/клуб.» Робокасса на витрине не затрагивается. Подробно — payments §3.1.2, публичная СБП-страница — payments §3.4.1.
