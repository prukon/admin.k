Ученик ↔ группы: связь M:N (team_user)

Файл: /docs/documentation/student-team-membership.html · миграция: 2026_06_20_160000_create_team_user_table.php
Кратко. У ученика (роль user) может быть несколько групп одновременно. Источник истины — pivot team_user (partner_id, team_id, user_id, timestamps, UNIQUE (team_id, user_id)). Колонка users.team_id сохранена как архив: при создании через factory/seed дублируется в pivot один раз; при изменениях групп в CRM не обновляется.

1) Модель данных

ОбъектНазначение
team_userАктуальная принадлежность ученика к группам партнёра
users.team_idLegacy-снимок (только create/backfill); не использовать в бизнес-логике CRM
contracts.group_idСнимок группы конкретного договора (не pivot ученика)
User::teams(), Team::students()Eloquent M:N через team_user

1.1) Сервисы и хелперы

  • App\Services\TeamUserSyncService — syncTeamsForStudent, attachTeamForStudent, detachAllTeamsForStudent, detachTeamFromAllStudents (при удалении группы), teamIdsForStudent, teamTitlesLabel (строка через запятую), syncLegacyTeamColumnToPivot (только create).
  • App\Support\UserPriceTeamMembership — проверка членства ученика в группе для users_prices и resolver оплаты; primaryTeamIdForStudent — первая группа в pivot.
  • App\Support\UserTeamQuery — фильтры отчётов и подзапрос названий групп без дублей строк.
  • Scope User::filterByStudentTeam($partnerId, $teamId) — ученик виден, если состоит в выбранной группе (none = без групп в pivot).

1.2) Удаление группы (DELETE /admin/team/{team})

Soft delete группы (teams.deleted_at) + отвязка всех учеников в pivot: TeamUserSyncService::detachTeamFromAllStudents($teamId, $partnerId) (TeamController@destroy). Legacy-колонка users.team_id при этом не обнуляется — актуальная принадлежность только в team_user. Строки users_prices с team_id удалённой группы в БД остаются (история начислений); новые начисления для soft-deleted группы не создаются. Подробнее — admin-teams § удаление.

3) Журнал расписания (/schedule)

См. schedule-journal.

ЭлементПоведение
Строка ученикаПод ФИО — подпись со списком групп (через запятую).
Фильтр группыУченик показывается, если состоит в выбранной группе (pivot).
Модалка «Управление группами»Multiselect + POST /schedule/user/{user}/sync-teams (SyncScheduleUserTeamsRequest, поле team_ids). Пустой массив — снять со всех групп.
POST …/set-groupLegacy attach одной группы / detach all (team_id пустой). UI журнала использует sync-teams.
GET …/cell-contextteam_ids, teams_label, context_team_id — для дефолтного тренера и подписи в модалке ячейки.
GET user-schedule/infouser.team_ids, user.team_titles; дни недели группы — по context_team_id (фильтр журнала или query).
Право schedule.view достаточно для изменения групп из журнала (отдельное users.group.update в этом UI не проверяется). Изменение групп ученика в карточке /admin/users по-прежнему требует users.group.update.

4) Договоры (/client-contracts)

См. contracts.

  • GET /client-contracts/user-group?user_id= — список групп ученика из pivot.
  • GET /client-contracts/users-search — поля team_ids, groups[] для модалки создания.
  • POST /client-contracts (ContractStoreRequest):
    • 0 групп у ученика → contracts.group_id = null;
    • 1 группа → подставляется автоматически;
    • 2+ группы → обязателен group_id из списка групп ученика.
  • Список договоров (DataTables): колонка «Группа» — contracts.group_id → teams.title (снимок на момент создания).
  • Карточка договора: поле «Группа» — все текущие группы ученика из pivot (TeamUserSyncService::teamTitlesLabel).

Фильтр объектаУченик попадает, если хотя бы одна его группа привязана к объекту (teams.location_id). См. location-team-bindings.

6) Личный кабинет и «Моя группа»

СтраницаПоведение
/account-settings/user/editПоле «Группы» — только чтение (readonly + disabled), значения из pivot через $user->teams. Подпись: «Изменение групп доступно только администратору CRM». В AccountUpdateRequest / PATCH нет team_id. Редактирование — /admin/users (users.group.update). См. parents-and-family-cabinet §4.5.
/my-group (myGroup.view)Если несколько групп — select «Выберите группу для просмотра»; GET /my-group/data?team_id= возвращает одногруппников выбранной группы.
«Мои документы»Read-only подпись групп ученика (pivot).

7) Контекст группы (context_team_id)

Когда у ученика несколько групп, часть UI/API принимает явный контекст одной группы:

  • Журнал — фильтр «Группа» на странице или data-context-team-id на ячейке;
  • cell-context, user-schedule/info — query context_team_id;
  • Дефолтный тренер «Посетил» — тренер группы из контекста (team_trainer), не первой попавшей группы без контекста.
