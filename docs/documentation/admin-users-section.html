Раздел CRM «Пользователи»: вкладки по ролям

Файл: /docs/documentation/admin-users-section.html · URL: /docs/documentation/admin-users-section · оглавление: /doc

1) Назначение

В админке пункт меню «Пользователи» ведёт на /admin/users. Внутри раздела — горизонтальные вкладки (partial resources/views/admin/users/_users_section_tabs.blade.php), по одной на каждую роль партнёра (кроме superadmin).
Каждая вкладка — отдельная страница со списком, добавлением и редактированием пользователей только этой роли (паттерн как у тренеров).

Состав вкладок и подписи задаёт сервис App\Services\Users\UsersSectionTabsResolver (порядок — roles.order_by).

2) Вкладки, URL и права

Вкладка (UI)Роль (roles.name)URL CRMВидимость вкладкиДокументация
Пользователи user (ученик) /admin/users users.role.update admin-users
Тренеры trainer /admin/trainers trainers.view admin-trainers
Администраторы admin /admin/administrators users.role.update admin-role-staff §3
{label} кастомной роли несистемная роль партнёра (is_sistem = 0) /admin/roles/{name} users.role.update admin-role-staff §4
Не путать: вкладка «Пользователи» показывает только учеников (role_id системной роли user). Администраторы, тренеры и кастомные роли — на своих вкладках. Журнал /schedule по-прежнему фильтрует только активных учеников — см. schedule-journal §2.1.

2.1) Какие роли попадают во вкладки

  • Все роли партнёра из roles: системные + привязанные через partner_role, кроме superadmin.
  • У обычного админа во вкладки попадают только роли с is_visible = 1 (как в матрице «Настройки → Права и роли»). У superadmin фильтр is_visible не применяется.
  • Новая кастомная роль из «Настройки → Права и роли» автоматически появляется как вкладка (если у актора есть users.role.update).
  • На маршрут /admin/roles/{name} нельзя попасть для зарезервированных имён user, admin, trainer, superadmin — ответ 404 (для них есть отдельные URL).

2.2) Меню сайдбара

Отдельного пункта «Тренеры» в сайдбаре нет: одна ссылка /admin/users («Пользователи»). Переход между ролями — только вкладками внутри раздела.

3) Общий UI

  • Заголовок страницы раздела: Пользователи (h4).
  • На каждой вкладке — toolbar (добавить, фильтры, колонки), DataTables через KidsCrmDataTable.create — см. reusable-ui-partials.
  • На вкладке «Пользователи» (/admin/users) дополнительно — кнопка «Импорт» при праве users.import (см. admin-users §8).
  • Trait контроллеров: App\Http\Controllers\Admin\Concerns\RendersUsersSectionTabs передаёт в view usersSectionTabs и activeTab.

4) Создание ученика: фиксированная роль

На /admin/users и при создании клиента из заявок с сайта (модалка §5.5.2, не отдельный createUser) селект роли скрыт: передаётся скрытое поле role_id = id роли user. Создать администратора или сотрудника с кастомной ролью через эту модалку нельзя — только через соответствующую вкладку.
