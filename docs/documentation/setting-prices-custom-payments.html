Установка цен → Дополнительные платежи

Файл: /docs/documentation/setting-prices-custom-payments.html. Страница CRM: /admin/setting-prices/custom-payments (вкладка custom_payments в admin/SettingPrices/index).
Группа платежа: с 2026‑06 у каждой записи user_custom_payment обязателен team_id. Группа определяет юр. лицо и маршрут выплаты T‑Bank при оплате (PayableTeamResolver → payables.meta.team_id). Подробно — payments §2.3.

1) Права доступа

PermissionНазначение
setPrices.viewРодительский middleware раздела «Установка цен» (обязателен для всех URL вкладки)
setPrices.customPayments.viewВкладка «Дополнительные платежи», все endpoint'ы ниже, ручная отметка оплаты (setting-prices.custom-payments.manual-paid)

Гость — 302/401/403. Без setPrices.view — 403 на весь раздел. Без setPrices.customPayments.view — 403 только на endpoint'ы доп. платежей. С правами — ожидаемые 200 / 302 (не 500, не пустой 200 на store).

2) Endpoint'ы

МетодURLRoute nameКонтракт
GET /admin/setting-prices/custom-payments admin.settingPrices.customPayments HTML вкладки; view admin.SettingPrices.index, activeTab=custom_payments
GET /admin/setting-prices/custom-payments/data admin.settingPrices.customPayments.data DataTables JSON: поля строки включают team_label (название группы или «—»)
GET /admin/setting-prices/custom-payments/users-search?q= admin.settingPrices.customPayments.users-search Select2: { results: [{ id, text }] }
GET /admin/setting-prices/custom-payments/teams-for-user?user_id= admin.settingPrices.customPayments.teams-for-user Select2 групп ученика: { results: [{ id, text }] }; пустой массив при user_id ≤ 0 или чужом партнёре
POST /admin/setting-prices/custom-payments admin.settingPrices.customPayments.store JSON { success: true, custom_payment: { …, team_id } }; валидация — 422 + errors (в т.ч. team_id). Ответ всегда JSON (не redirect), в т.ч. без X-Requested-With
POST /admin/setting-prices/custom-payments/{id}/manual-paid setting-prices.custom-payments.manual-paid Тот же permission setPrices.customPayments.view; JSON success / 404

3) UI

  • Таблица #custom-payments-table — колонки: ученик, Группа, период, сумма, комментарий, статус, действия. JS: resources/js/setting-prices-custom-payments.js.
  • Модалка «Добавить дополнительный платёж»: Select2 ученика → после выбора подгружаются группы через teams-for-user; поле team_id обязательно.
  • Валидация: UserCustomPaymentStoreRequest — team_id required, exists в teams партнёра, ученик состоит в группе (сообщение под полем «Ученик не состоит в выбранной группе.»).

4) База данных

  • Колонка user_custom_payment.team_id (nullable FK → teams) — миграция 2026_06_29_150000_add_team_id_to_custom_payment_and_lesson_package_assignments.php.
  • Backfill для старых строк — 2026_06_29_160000_backfill_team_id_on_payment_assignments.php, сервис PaymentAssignmentTeamBackfill (primary group ученика).

5) Связь с оплатой

При Init (custom_payment_fee) PayableTeamResolver читает user_custom_payment.team_id, пишет в payables.meta.team_id и payment_intents.meta. Без группы на записи — 422 при старте оплаты. См. payments §3.3.

На консоли (/cabinet) селект «Выбор группы» не фильтрует список доп. платежей — группа зафиксирована в админке при создании записи. См. dashboard-cabinet §4.5.
