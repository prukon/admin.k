Настройки → Платёжные системы (/admin/settings/paymentSystem)

Файл: /docs/documentation/settings-payment-systems.html · оплаты: payments · T‑Bank flow: tbank · комиссии и автовыплата: tbank §5
Изменение архитектуры (2026‑06): ключи терминала T‑Bank (eacq + e2c) хранятся один раз на всю платформу (payment_systems.partner_id IS NULL). Робокасса по-прежнему настраивается отдельно для каждого партнёра. Автовыплата и задержка после оплаты перенесены в tinkoff_commission_rules (партнёр + метод), а не в payment_systems и не в глобальную настройку settings.

3) Robokassa (per-partner)

  • Запись в БД: payment_systems(partner_id = текущий партнёр, name = robokassa).
  • Поля в settings (зашифрованный JSON): merchant_login, password1, password2, password3; флаг test_mode на модели.
  • Доступность на витрине: PaymentService::isRobokassaAvailable() — запись есть и is_enabled = true.
  • Форма: модалка «Подключить», submit через $.ajax на payment-systems.store.

4) T‑Bank — глобальный терминал платформы

Одна запись на платформу: payment_systems(partner_id = NULL, name = 'tbank'). Старые строки payment_systems(name=tbank, partner_id=…) в коде не читаются — их нужно удалить вручную после переноса ключей. Fallback на .env / config/tinkoff.php для ключей терминала отключён.

Признак «подключено» в UI: аксессор PaymentSystem::is_connected — все четыре ключа заполнены.

4.2 Сервис чтения конфигурации

Единая точка для бизнес-кода и webhook'ов — App\Services\Tinkoff\TbankTerminalConfig:

  • globalRecord() — модель или null
  • isGloballyActive() — запись есть, is_enabled и is_connected
  • paymentConfig() / e2cConfig() — ключи и base URL (test/prod); бросают RuntimeException, если терминал не настроен
  • tryPaymentConfig() / tryE2cConfig() — без исключения, null если не настроено

4.3 Доступность T‑Bank у партнёра на витрине

PaymentService::isTbankAvailable($partner, ?Team $team) возвращает true, если одновременно:

  1. TbankTerminalConfig::isGloballyActive() — глобальный терминал включён и ключи заполнены;
  2. есть ShopCode у юр. лица, выбранного LegalEntityResolver для партнёра/группы, или (legacy fallback) заполнен partners.tinkoff_partner_id;
  3. на витрине дополнительно проверяются права пользователя: payment.method.tbankCard / payment.method.tbankSBP.

Для payable с контекстом группы: isTbankAvailableForPayable(). См. admin-legal-entities §1.1.

СБП: isTbankSbpAvailable($partner, $amountCents) — то же + сумма от 10 ₽ (1000 коп.).

4.4 UI на странице настроек

  • Карточка «T‑Bank (мультирасчёты)» с подписью «Терминал платформы (общий для всех партнёров)».
  • Чекбокс «T‑Bank включён на платформе» (is_enabled).
  • Модалка подключения: eacq-ключи, e2c-ключи, тестовый режим.
  • Переменная Blade $tbank всегда из PaymentSystem::globalTbank(), не из списка $paymentSystems текущего партнёра.

4.5 Сохранение (AJAX и safety-net)

  • Формы Robokassa и T‑Bank: e.preventDefault() + $.ajax POST payment-systems.store.
  • AJAX / JSON (X-Requested-With или Accept: application/json): ответ { "status": "success", "message": "…" }, статус 200.
  • Non-AJAX POST (нативная отправка формы без JS): redirect 302 на /admin/settings/paymentSystem с flash status; запись в БД создаётся/обновляется. Пустой 200 не возвращается.
  • Ошибки валидации: redirect back с ошибками в сессии (302).

5) Связь с комиссиями и автовыплатой

Ключи терминала — здесь (глобально). Комиссии, автовыплата и задержка после оплаты — в /admin/settings/tbank-commissions (tinkoff_commission_rules.auto_payout_enabled, auto_payout_delay_hours per партнёр + метод). Интервал джобы отложенных выплат — в модалке «Настройки выплат» на странице комиссий (settings.tinkoff_payout_scheduled_interval_minutes).

6) Миграция с per-partner T‑Bank (ручные шаги)

  1. Создать глобальную запись payment_systems(partner_id=NULL, name=tbank) через UI или SQL.
  2. Перенести ключи из любой рабочей per-partner строки в глобальную.
  3. Выставить is_enabled и при необходимости test_mode.
  4. Удалить устаревшие payment_systems(name=tbank, partner_id IS NOT NULL).
  5. Для каждого партнёра настроить автовыплату в правилах комиссий (/admin/settings/tbank-commissions/{id}/edit).
  6. Убедиться, что у партнёра зарегистрировано хотя бы одно юр. лицо с ShopCode в справочнике «Юр. лица» (или временно заполнен legacy partners.tinkoff_partner_id).

Автоматической миграции данных в релизе нет — по решению владельца платформы.
