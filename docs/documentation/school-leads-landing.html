Страница заявки партнёра (брендированный лендинг)

Файл: /docs/documentation/school-leads-landing.html · оглавление: /doc · см. также заявки с сайта (виджет iframe и CRM)
Назначение: отдельная публичная страница с полной формой записи для конкретного партнёра (школы). Партнёр раздаёт ссылку в рекламе, соцсетях или QR-коде. Заявки сохраняются в ту же таблицу school_leads, что и заявки из iframe-виджета, с пометкой source = landing.
Не путать с iframe-виджетом (/widget/{widgetKey}) — это короткая форма для встраивания на сайт партнёра; см. school-leads-widget §4.

1) Два канала сбора заявок школы

КаналПубличный URLКлючФормаschool_leads.source
Страница заявки (этот документ) /lead/{landingKey} partner_widgets.landing_key Полная форма: родитель, ребёнок, район, вид спорта (если есть справочник), услуга, особенности landing
Виджет iframe /widget/{widgetKey} partner_widgets.widget_key Короткая форма: имя, телефон, согласие widget (по умолчанию)

Оба канала используют одну запись partner_widgets на партнёра и одни и те же уведомления (email / Telegram). CRM — раздел /admin/school-leads.

2) Привязка к партнёру (безопасность)

  • Не передавать partner_id с фронта — только сервер определяет партнёра по landing_key в URL.
  • Ключ — 48 символов ([A-Za-z0-9]{48}), уникален в partner_widgets.landing_key.
  • Создаётся вместе с виджетом: PartnerWidgetService::ensureForPartner (при создании партнёра и при первом обращении в CRM).
  • Страница доступна, если is_landing_active = true. Иначе — 404 на все endpoint'ы лендинга.
  • Все справочники (районы, виды спорта, услуги) фильтруются по partner_id виджета; чужие location_id / sport_type_id / team_id отклоняются валидацией (422).

3.1) Доступ без авторизации

Страница и API лендинга публичные: гость и авторизованный пользователь CRM получают 200 при валидном запросе. Middleware auth на маршруты не навешен.

Покрытие: tests/Feature/Public/SchoolLeadLandingPublicAccessFeatureTest.php.

4) Поля формы

4.1) Законный представитель (обязательные)

  • Фамилия, имя, отчество
  • Телефон, email

4.2) Ребёнок (обязательные)

  • Фамилия, имя, отчество
  • Дата рождения (должна быть в прошлом)

4.3) Особенности (чекбоксы да/нет)

  • is_individual_traits — индивидуальные особенности воспитанника
  • is_on_medical_register — состоит на учёте у медицинских специалистов
  • is_with_disability — наличие инвалидности

4.4) Район, вид спорта и услуга

  • Район (location_id) — обязателен. Справочник: активные locations партнёра (is_enabled = true).
  • Вид спорта (sport_type_id) — необязателен. Селект показывается только если у партнёра есть хотя бы один активный вид спорта (sport_types.is_enabled = true). Справочник ведётся в /admin/sport-types. После выбора района селект активируется; при смене района или вида спорта список услуг перезагружается.
  • Услуга (team_id) — необязательна. Список подгружается AJAX после выбора района (и опционально вида спорта); фильтр через TeamLocationAvailabilityService и SchoolLeadLandingService::teamsForLocation() (см. локации ↔ группы).
  • Если переданы и sport_type_id, и team_id, группа должна иметь тот же teams.sport_type_id; иначе 422 на поле team_id.
  • Чекбокс needs_contact_help — «Не могу определиться с секцией, прошу со мной связаться»: при включении team_id сбрасывается и не обязателен.

4.5) Прочее

  • Комментарий (comment) — необязателен
  • Согласие на обработку ПДн (consent_accepted) — обязательный чекбокс; policy_url для лендинга = null (без ссылки на политику)
  • UTM и tracking: utm_*, page_url, referrer — скрытые поля, заполняются JS

6) Отображение на странице

  • Заголовок страницы: «Записаться / на регулярные тренировочные занятия».
  • Заголовок секции формы: «Район, вид спорта и услуга» (если справочник видов спорта пуст — блок «Вид спорта» не рендерится).
  • Название партнёра — partners.title (поле «Наименование» в настройках организации); если пусто — organization_name (Partner::landingDisplayName()).
  • Цвета шапки — фиксированная зелёная тема в CSS view (кастомизация бренда в CRM — запланирована).

8) Уведомления

После успешного POST /lead/…/submit вызывается SchoolLeadNotificationService::notify — те же каналы, что у виджета:

  • Email — admin'ы партнёра + partners.email
  • Telegram — в partners.school_leads_telegram_chat_id; для лендинга текст дополнен (ребёнок, район, услуга, флаг «помочь с секцией»)

9) Модель partner_widgets

Миграция: 2026_05_29_100000_add_landing_key_to_partner_widgets_table.php.

  • widget_key — для iframe-виджета (/widget/…)
  • landing_key — для страницы заявки (/lead/…)
  • is_active — виджет iframe
  • is_landing_active — страница заявки
