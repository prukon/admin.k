Переиспользование UI-блоков (partial / include)

Файл: /docs/documentation/reusable-ui-partials.html · Blade: resources/views/partials/
Общие фрагменты интерфейса вынесены в partial'ы с директивой @once (стили/скрипты подключаются один раз на страницу через @push('styles') / @push('scripts')). На новой странице достаточно @include('…') и вызова JS-API из inline-скрипта страницы.

1) Шапка списка: toolbar + фильтры + «Поля списка»

Единый паттерн карточки над таблицей: заголовок слева, справа кнопки «Добавить» (по праву), «Фильтры» (collapse), «Колонки» (dropdown с чекбоксами data-column-key).

CSS

  • resources/css/admin-list-toolbar.css — классы payments-report-toolbar, payments-report-surface, payments-report-filters-toggle, payments-report-columns-menu и др.
  • Подключение на странице: @vite(['resources/css/admin-list-toolbar.css']).

2) Hover-список в ячейке таблицы

Partial: resources/views/partials/ui/hover-list-dropdown.blade.php

Краткая подпись в ячейке + полный список пунктов в Bootstrap Tooltip при наведении. Стиль совпадает с подсказками «Установка цен → По месяцам» (ulp-assignment-paid-tooltip).

Подключение

@include('partials.ui.hover-list-dropdown')

JS API (window.KidsCrmHoverListDropdown)

МетодНазначение
renderCell(shortLabel, itemsArray) HTML ячейки DataTables: shortLabel — текст в таблице; itemsArray — массив строк для списка в tooltip (если 0–1 пункт, tooltip не показывается).
init(rootElement) Инициализация tooltip'ов внутри контейнера (обычно #…-table или document). Вызывать после draw.dt.
dispose(rootElement) Снять tooltip'ы перед перерисовкой (опционально).

Пример в колонке DataTables

columns: [{
  data: 'teams_label',
  render: function (data, type, row) {
    if (window.KidsCrmHoverListDropdown) {
      return KidsCrmHoverListDropdown.renderCell(data, row.teams_titles || []);
    }
    return data || '';
  }
}],
…
table.on('draw.dt', function () {
  if (window.KidsCrmHoverListDropdown) {
    KidsCrmHoverListDropdown.init(document.getElementById('locations-table'));
  }
});

Где используется

  • /admin/locations — колонка «Группы» (teams_label + teams_titles из JSON /admin/locations/data).

Прямой аналог tooltip без partial — inline в admin/SettingPrices/users.blade.php (ulp-assignment-paid-tooltip); для таблиц предпочтителен переиспользуемый partial.

3) Select2 multiselect групп

Partial: resources/views/partials/select2/teams-multiselect.blade.php

Multiselect на Select2 (theme bootstrap-5): чекбоксы в dropdown, сводка «первая группа, ещё N шт.» при 3+ выбранных, dropdown не закрывается при клике в поиск.

Подключение

@include('partials.select2.teams-multiselect')

<select name="team_ids[]"
        class="form-select js-teams-multiselect-select"
        multiple
        data-placeholder="Выберите группы">
  @foreach($teamOptions as $team)
    <option value="{{ $team->id }}">{{ $team->title }}</option>
  @endforeach
</select>

Select2 подключается в layout layouts/admin2.blade.php (select2.full.min.js + bootstrap-5 theme).

JS API (window.KidsCrmTeamsMultiselectSelect2)

  • init($select, { dropdownParent }) — инициализация; для модалок задавать dropdownParent: $('#…Modal').
  • setValues($select, ids) — заполнить при открытии edit.
  • reset($select) — очистить create-модалку.
  • markInvalid($select) / clearInvalid($select) — подсветка ошибок валидации (team_ids).

Где используется

  • /admin/locations — поля locationCreateTeamIds, locationEditTeamIds (только при locations.manage и непустом списке групп партнёра).

4) Русификация Select2

Partial: resources/views/partials/select2/ru.blade.php

Возвращает объект локализации (не JSON-строку) для опции language при вызове .select2({ … }).

$('.my-select').select2({
  theme: 'bootstrap-5',
  language: @include('partials.select2.ru')
});

Тексты: «Ничего не найдено», «Поиск…», «Введите ещё N символ(а/ов).», «Можно выбрать не более …» и т.д.

Partial teams-multiselect подключает эту локализацию автоматически. На других страницах — явный @include('partials.select2.ru') (отчёты, dashboard, договоры, абонементы, custom-payments и др.).

5) Русификация DataTables

Partial: resources/views/partials/datatables/ru.blade.php

Возвращает JSON-объект для опции language jQuery DataTables.

$('#my-table').DataTable({
  serverSide: true,
  language: @include('partials.datatables.ru'),
  …
});

Строки: «Обработка…», «Поиск…» (placeholder), «С _START_ до _END_ из _TOTAL_ записей», «Записи отсутствуют.», «В таблице отсутствуют данные», aria-подписи сортировки. Кнопки paginate (first/previous/…) — пустые строки (иконки/разметка задаются темой таблицы).

Где используется

  • /admin/locations, /admin/trainers, /admin/setting/logs, contract-templates/index
  • вкладка назначений абонементов (lessonPackages/tabs/assignments.blade.php)
  • и другие server-side таблицы с тем же паттерном

6) Чек-лист при добавлении новой страницы

  1. Подключить admin-list-toolbar.css, если нужна шапка как у отчётов/списков.
  2. Для server-side таблицы — language: @include('partials.datatables.ru').
  3. Для Select2 — language: @include('partials.select2.ru') (layout admin2 уже грузит JS/CSS Select2).
  4. Для колонки «много значений, коротко в ячейке» — @include('partials.ui.hover-list-dropdown') + renderCell / init на draw.dt.
  5. Для multiselect групп в модалке — @include('partials.select2.teams-multiselect') + класс js-teams-multiselect-select.
