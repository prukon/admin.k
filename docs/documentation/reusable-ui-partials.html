Переиспользование UI-блоков (partial / include)

Файл: /docs/documentation/reusable-ui-partials.html · Blade: resources/views/partials/
В layout layouts/admin2.blade.php через Vite глобально подключены модули admin-списков: resources/css/kids-tooltip.css, resources/css/datatables-columns.css, resources/js/kids-tooltip.js, resources/js/kids-datatable.js. На новой admin-странице с server-side таблицей достаточно вызвать KidsCrmDataTable.create(...) — отдельные @include для tooltip/DataTables JS не нужны. Partial'ы с @once по-прежнему используются для HTML-фрагментов (иконка-подсказка, Select2) и документации в коде.

2) Tooltip / hover (единый модуль)

Файлы: resources/js/kids-tooltip.js, resources/css/kids-tooltip.css.

Единый Bootstrap Tooltip в стиле ulp-assignment-paid-tooltip: текст с ellipsis, списки в ячейках, иконки-подсказки в формах.

JS API (window.KidsCrmTooltip)

МетодНазначение
renderText(text, options?) Обрезанный текст (класс dt-cell-ellipsis). Tooltip и cursor: help включаются в init только если текст не помещается в ячейку (scrollWidth > clientWidth). Пустое значение → «—».
renderLink(text, { linkClass, extraAttrs, href?, emptyHtml? }) Ссылка с ellipsis внутри; tooltip на <span>, чтобы не конфликтовать с data-bs-toggle="modal" на <a>. Опция href — реальный URL или javascript:void(0); (по умолчанию), если переход обрабатывается JS (модалка, delegated click). Стили ссылок в ячейке — kids-tooltip.css (синий цвет, подчёркивание).
renderList(shortLabel, itemsArray, options?) Краткая подпись + список пунктов в HTML-tooltip. При 0–1 пункте tooltip не показывается (опция minItemsForHover).
init(root, { scopes? }) Инициализация tooltip'ов в контейнере. Scopes: list, text, hint, manualPaid, generic (по умолчанию — первые три: list, text, hint).
dispose(root, { scopes? }) Снять tooltip'ы (например, перед перерисовкой Select2).
bindDataTable(tableElement) Подписка на draw.dt + init (после requestAnimationFrame, чтобы корректно определить обрезку текста). Вызывается автоматически из KidsCrmDataTable.create.
escapeHtml(text) Экранирование для безопасного HTML.
decodeHtmlEntities(text) / normalizeDisplayText(text) Снятие HTML-сущностей из JSON (Yajra DataTables экранирует текст в ответе). Перед renderText строка нормализуется, чтобы в ячейке не показывались литералы вроде &quot;.

Пустые текстовые значения: <span class="dt-cell-empty text-muted">—</span> (единый прочерк во всех колонках типа text).

Иконка-подсказка в форме

Partial (только HTML): resources/views/partials/ui/tooltip-hint.blade.php

@include('partials.ui.tooltip-hint', [
    'title' => 'Текст подсказки',
    'placement' => 'top',
])

Инициализация hint'ов на DOMContentLoaded и при открытии модалок — через KidsCrmTooltip.init(..., { scopes: ['hint'] }) (глобально в kids-tooltip.js).

Scope generic — для элементов с data-bs-toggle="tooltip" и data-bs-title (или title), если tooltip создаётся вне пресета DataTable (например, на страницах расписания, в модалках цен).

Важно: единственное место создания new bootstrap.Tooltip(...) в проекте — модуль kids-tooltip.js. На страницах не вызывать Bootstrap Tooltip напрямую.

Исключение из правила «tooltip только при обрезке»: элементы с js-dt-cell-ellipsis-tooltip, но без класса dt-cell-ellipsis (например иконка договора в /admin/users) — tooltip с явным data-bs-toggle="tooltip" и title включается всегда.

3) DataTables: пресет KidsCrmDataTable

Файл: resources/js/kids-datatable.js. Документация в коде: partials/datatables/kids-datatable.blade.php.

Объединяет: декларативные колонки, ширины (datatables-columns.css), tooltip'ы, настройки видимости колонок (columns-settings).

        d.status = $('#filter-status').val() || '';
      },
    },
    order: [[1, 'asc']],
    language: @include('partials.datatables.ru'),
  },
  columns: [
    { key: 'id', type: 'id', data: 'id' },
    { key: 'name', type: 'text', data: 'name' },
    { key: 'address', type: 'text', data: 'address' },

const table = dtApi.table;

3.2) Кликабельные названия в таблицах (единый паттерн, июнь 2026)

На admin-страницах с KidsCrmDataTable колонка «название» / «имя» / «ФИО» оформляется как type: 'link' через KidsCrmTooltip.renderLink. Визуально — синяя подчёркнутая ссылка с ellipsis и tooltip при обрезке (resources/css/kids-tooltip.css, селектор td.dt-col-text > a:not(.btn)).

Два сценария поведения по клику:

СценарийКолонкаCSS-классДействие по клику
Модалка редактирования (без перезагрузки страницы) type: 'link' + linkClass + linkAttrs(row) Свой класс (js-district-edit, edit-team-link, edit-lead …) Delegated handler на странице: AJAX → JSON → открыть Bootstrap-модалку. Атрибуты строки — обычно data-id или data-template-id.
Переход на другую страницу type: 'link' + navigate: true и href(row) или ручной linkAttrs с data-href js-dt-nav-link (добавляется пресетом при navigate или вручную) KidsCrmDataTable.bindNavLinks(tableElement) — preventDefault + window.location.assign(href). URL берётся из data-href или из href, если это не javascript:void(0);.

Пример: модалка (районы)

{
  key: 'name',
  type: 'link',
  data: 'name',
  className: 'dt-col-text',
  linkClass: 'js-district-edit',
  linkAttrs: function (row) {
    return 'data-id="' + row.id + '"';
  },
}

На той же странице — обработчик $(document).on('click', '.js-district-edit', …) → GET /admin/districts/{id} (JSON) → модалка edit.

Пример: навигация (договоры)

{
  key: 'user_name',
  type: 'link',
  data: 'user_name',
  linkClass: 'js-dt-nav-link',
  linkAttrs: function (row) {
    return 'data-href="/client-contracts/' + row.id + '"';
  },
}
// после KidsCrmDataTable.create:
KidsCrmDataTable.bindNavLinks(document.getElementById('contracts-table'));

Альтернатива без ручного data-href: navigate: true, href: function (row) { return '/client-contracts/' + row.id; } — пресет сам добавит js-dt-nav-link.

Условная ссылка (только при праве manage)

Если редактирование недоступно, колонка остаётся type: 'text' (или text-long) — без класса-обработчика. Пример: объекты (locations.manage), виды спорта (sport_types.manage).

CSS ширин колонок

Файл: resources/css/datatables-columns.css. Класс dt-columns-managed на <table> добавляется автоматически в KidsCrmDataTable.create.

  • Режим: table-layout: auto, таблица на 100% ширины карточки — колонки сужаются под короткий контент.
  • № / actions: width: 1% + min-width — узкие служебные колонки.
  • count / money / badge / sort / image / icon: width: 1%, white-space: nowrap — по ширине значения; у image фиксированный размер через inline width/height на <img>. У money сумма выравнивается вправо через .dt-col-money-value; прочерк «—» — слева, как у text.
  • text / list / schedule: width: 1% (shrink-to-content) + cap --dt-col-ellipsis-max (18rem) + ellipsis; полный текст — в tooltip через KidsCrmTooltip только если строка обрезана.
  • datetime: класс dt-col-text--wrap, разметка pay-cell-datetime (стили в admin-list-toolbar.css).
  • inline-actions: узкая колонка под две кнопки (dt-col-inline-actions).
  • details-control (chevron в LTV / «Платежи по месяцам»): width: 1%, компактная кнопка .details-control-btn.
  • Вертикальное выравнивание: vertical-align: middle на всех th/td managed-таблиц.

Не задавайте inline min-width / width в <th> — ширина только через dt-col-* в определении колонок.

Горизонтальный скролл

Поиск DataTables (.dataTables_filter) и блок «Показать N записей» остаются на ширине карточки и не уезжают при горизонтальном скролле.

  • У .dataTables_wrapper — overflow-x: visible.
  • Скролл только у области таблицы: обёртка .kids-dt-scroll-x (добавляется в init.dt через KidsCrmDataTable.ensureTableScrollHost) или встроенный .dataTables_scroll при scrollX / fixedColumns.

Пагинация: одна страница

По умолчанию пресет в drawCallback скрывает только блок .dataTables_paginate (prev/next, номера страниц), если page.info().pages <= 1. Строки «Показано X–Y из Z» и выпадающий «Показать N записей» не скрываются. При смене «Показать 10 / 20 / …» пересчёт выполняется на каждой отрисовке (например, 15 записей при 20 на странице — пагинация скрыта; при 10 на странице — видна).

Отключить на конкретной таблице: dataTable: { hidePaginationWhenSinglePage: false }.

Partial: resources/views/includes/logModal.blade.php. Подключается на страницах users, teams, schedule, договоров, справочников, заявок, отчётов и др. Кнопка в toolbar — иконка fa-clock-rotate-left, при открытии showLogModal(url) загружает GET …/logs-data (см. audit-my-logs §5.1).

  • Таблица #logsTable — KidsCrmDataTable.create без блока columnsSettings (набор колонок фиксирован).
  • Инициализация отложенная: при первом shown.bs.modal на #historyModal, URL берётся из window.showLogModal(routeName).
  • На вкладке «Настройки → Логи» — отдельная страница с полным набором фильтров (см. settings-logs), не модалка.

Вложенные (дочерние) таблицы в отчётах

В LTV и «Платежи по месяцам» при раскрытии строки детализация платежей использует обычный $('#…').DataTable({ serverSide: true }), не KidsCrmDataTable.create — у дочерней таблицы нет dropdown «Колонки». Backend поддерживает два формата одного endpoint: legacy JSON без draw и server-side DataTables с draw + meta_payments_count / meta_sum_total (см. reports-admin §4–5).

4) Multiselect: модалки (Select2) и фильтры (кастомный UI)

Два независимых компонента. Не смешивать: в модалках — chip-теги на Select2; в панелях фильтров — trigger как form-select без Select2.

4.1) Filter multiselect (filter-multiselect)

@include('partials.select2.filter-multiselect')

<select class="form-select js-filter-multiselect-select"
        id="sl-filter-status"
        name="status_ids[]"
        multiple
        data-placeholder="Выберите статусы">
  …
</select>

JS API: window.KidsCrmFilterMultiselectSelect2 — init, setValues, rebuild, reset, markInvalid / clearInvalid.

Где используется:

  • /admin/school-leads — фильтр «Статус» (status_ids[]).

4.2) Select2 multiselect для модалок (generic-multiselect)

Partial-обёртки: teams-multiselect.blade.php, locations-multiselect.blade.php → внутри generic-multiselect.blade.php.

Multiselect на Select2 (theme bootstrap-5): чекбоксы в dropdown, chip-теги, сводка «первая группа, ещё N шт.» при 3+ выбранных, dropdown не закрывается при клике в поиск.

Подключение (модалка)

@include('partials.select2.teams-multiselect')

<select name="team_ids[]"
        class="form-select js-teams-multiselect-select"
        multiple
        data-placeholder="Выберите группы">
  @foreach($teamOptions as $team)
    <option value="{{ $team->id }}">{{ $team->title }}</option>
  @endforeach
</select>

Обобщённый multiselect с hover-list в chip: partials/select2/generic-multiselect.blade.php (сводка 3+ значений через KidsCrmTooltip.renderList / init / dispose). Класс на select: js-generic-multiselect-select.

Select2 подключается в layout layouts/admin2.blade.php (select2.full.min.js + bootstrap-5 theme).

JS API модалок (window.KidsCrmGenericMultiselectSelect2)

  • init($select, { dropdownParent }) — инициализация; для модалок задавать dropdownParent: $('#…Modal').
  • setValues($select, ids) — заполнить при открытии edit.
  • reset($select) — очистить create-модалку.
  • markInvalid($select) / clearInvalid($select) — подсветка ошибок валидации (team_ids).

Где используется (модалки)

  • /admin/locations — поля locationCreateTeamIds, locationEditTeamIds, администраторы (только при locations.manage и непустом списке групп партнёра).
  • /admin/districts — multiselect объектов в create/edit модалках.

7) Чек-лист: новая admin-страница со server-side таблицей

  1. Layout admin2 — модули tooltip/DataTable уже подключены; после изменений JS/CSS выполнить npm run build.
  2. Шапка списка — @vite(['resources/css/admin-list-toolbar.css']) + разметка toolbar (см. §1).
  3. Таблица — class="table … w-100 dt-columns-managed" (класс dt-columns-managed можно не дублировать: пресет добавит сам).
  4. Инициализация — KidsCrmDataTable.create('#…-table', { columnsSettings, dataTable, columns }) (см. §3).
  5. Dropdown «Колонки» — чекбоксы class="column-toggle" с data-column-key, совпадающим с columns[].key.
  6. Backend — GET/POST …/columns-settings с объектом columns (как на locations / sport-types / teams).
  7. Reload после фильтров — dtApi.reload({ keepPage: true }).
  8. Кликабельное название — type: 'link' + handler модалки или bindNavLinks (см. §3.2).
  9. Для Select2 в модалках — partial generic-multiselect (§4.2) + @include('partials.select2.ru') при необходимости.
  10. Для multiselect в панели фильтров — partial filter-multiselect (§4.1), KidsCrmFilterMultiselectSelect2; без Select2.
  11. Для иконки-подсказки в форме — @include('partials.ui.tooltip-hint', …) (§2).

Страницы вне пресета (по состоянию на 2026‑06): журнал расписания /schedule — client-side DataTable в schedule.js (намеренно, без server-side); личный отчёт ученика /user/report/payment — legacy inline DataTable. При миграции переводить на KidsCrmDataTable / KidsCrmTooltip по чек-листу §7.
