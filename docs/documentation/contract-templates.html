Шаблоны договоров (DOCX)

Файл: /docs/documentation/contract-templates.html · CRM: вкладка «Шаблоны» на /client-contracts или /client-contract-templates · см. договоры (CRM), заполнение родителем

1) Назначение

Партнёр загружает DOCX с плейсхолдерами {{ключ}}. Система извлекает ключи, формирует fields_schema (подписи, обязательность, prefill). При создании договора с creation_mode=template списывается 70 ₽, договор получает статус awaiting_client_fill, срок заполнения fill_expires_at (+7 дней, Contract::FILL_TTL_DAYS). Клиенту уходит email-приглашение и задача в личном кабинете (/account-settings/documents). PDF формирует родитель в кабинете; подпись — SMS (Подпислон) из кабинета, не из CRM.

2) Раздел «Документы» в CRM

На странице /client-contracts две вкладки: «Договоры» и «Шаблоны» (ContractsDocumentsSectionUiFeatureTest). Шаблоны также доступны по прямому URL /client-contract-templates (view contract-templates.index, activeTab=templates).

QueryДействие
?create=1Открыть модалку «Создать шаблон» (#createContractTemplateModal)
?edit={template_id}Открыть модалку редактирования (#editContractTemplateModal)
?email={template_id}Открыть модалку email (#editTemplateEmailModal)

Базовое право раздела: contracts.view. Маршруты с {template} дополнительно проходят middleware contract-template.partner (шаблон текущего партнёра).

Право contracts.templates.fillSortOrder.edit (скрытое, не в базовых ролях): колонка «Порядок» в редакторе полей и сохранение fill_sort_order в fields_schema. По умолчанию доступно только superadmin (Gate::before + явная выдача права). Backend: ContractTemplateService::normalizeFieldsInput() игнорирует fill_sort_order из запроса без этого Gate.

5) Модалки шаблонов

5.1) Создание (#createContractTemplateModal)

  • Поля: название, файл DOCX (обяз.), аккордеон «Переменные» (справочник пресетов).
  • После загрузки DOCX — таблица полей (fields-editor, id таблицы contract-template-fields-table).
  • Колонки: ключ, подпись, группа (родитель/ученик/система), обязательность, prefill; «Порядок» — только при @can('contracts.templates.fillSortOrder.edit').
  • Валидация: ContractTemplateStoreRequest.

5.2) Редактирование (#editContractTemplateModal)

  • Название, опционально новый DOCX (создаёт новую версию, старые договоры остаются на своей версии).
  • Редактор полей тот же; системные поля (contract_date, contract_id…) помечены и не удаляются.
  • PUT contract-templates.update не меняет email — только subject/body через отдельный endpoint.
  • Валидация: ContractTemplateUpdateRequest.

5.3) Email (#editTemplateEmailModal)

  • Открывается кнопкой «Email» в таблице или ?email= на index.
  • AJAX GET …/email → subject, body HTML, список плейсхолдеров, update_url.
  • PUT …/email — ContractTemplateUpdateEmailRequest.
  • Дефолты при пустых полях версии: ContractTemplateEmailDefaults.
  • Плейсхолдеры письма: {{child_full_name}}, {{partner_name}}, {{documents_url}}, {{fill_deadline}}, {{contract_id}} — подстановка в ContractInvitationEmailRenderer.
  • Отправка при создании договора: ContractClientFillInvitationMail (ContractCreationService::notifyClientAboutFill).

5.4) UI-заметки

Таблица полей шаблона использует id contract-template-fields-table (не fields-table), чтобы не конфликтовать с глобальными стилями модалки полей CRM. Горизонтальный скролл и чекбоксы «Обяз.» синхронизированы через CSS в fields-editor.blade.php.

6) Переменные DOCX и fields_schema

Единый формат ключей: snake_case, например {{parent_full_name}}. Справочник рекомендуемых переменных, русские подписи, prefill_source, группы и fill_sort_order по умолчанию — класс ContractTemplateVariablePresets.

Ключ DOCXВ форме родителя
parent_full_nameparent_lastname, parent_firstname, parent_middlename
child_full_namechild_lastname, child_firstname
child_birthdayполе даты; в PDF — d.m.Y

При загрузке DOCX для известных ключей автоматически подставляются подписи и prefill из пресетов; неизвестные ключи попадают в схему как кастомные поля.

7) Версии DOCX и архив

  • Каждый новый файл DOCX при store/update создаёт запись в contract_template_versions и обновляет current_version_id.
  • Уже созданные договоры ссылаются на contract_template_version_id на момент создания.
  • Архивирование шаблона (is_archived) скрывает его из Select2 при создании договора (ContractTemplate::active()).
