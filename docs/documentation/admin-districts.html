Районы (админка): /admin/districts

Файл: /docs/documentation/admin-districts.html · обзор: directories-hierarchy · объекты: admin-locations · лендинг: school-leads-landing
Глоссарий UI: в интерфейсе «Район» = модель District, «Объект» = модель Location.
Пункт сайдбара справочников (виден при districts.view, locations.view, groups.view или sport_types.view): при единственном districts.view подпись «Районы», при 2+ правах — «Справочники» (DirectoriesMenu). Вкладка «Районы» — третья: Группы → Объекты → Районы → Виды спорта ($activeTab = 'districts'). URL пункта меню — первая доступная вкладка по приоритету: группы → объекты → районы → виды спорта.
Все маршруты раздела — под middleware('can:districts.view'). Право districts.view покрывает просмотр и CRUD (отдельного manage нет).

2) Модель и БД

  • Таблица districts: partner_id, name, is_enabled, sort_order.
  • Уникальность: unique(partner_id, name) — после удаления имя можно использовать снова.
  • Связь: District hasMany Location (locations.district_id nullable).

3) Интерфейс

View: resources/views/admin/districts/index.blade.php, вкладки: admin/directories/_section_tabs.blade.php.

  • Toolbar — «Добавить», «История», «Фильтры», «Колонки».
  • Поля формы create/edit: name, is_enabled, sort_order.
  • Multiselect «Объекты» (location_ids[]) — в модалках создания и редактирования, при locations.view и непустом справочнике объектов. Select2 через partial generic-multiselect (см. reusable-ui-partials). При сохранении — полная замена списка объектов района через LocationDistrictSyncService: выбранным проставляется locations.district_id, снятым с района — null.
  • GET /admin/districts/{id} возвращает location_ids только при locations.view; без права ключ отсутствует.
  • Колонка «Объекты» — при locations.view и непустом справочнике: тип DataTables list, ключ locations_label (сортировка по locations_count). До 2 названий — через запятую; при 3+ — «Первый, еще N шт.» + tooltip (locations_label_full, locations_names). Число объектов — locations_count (withCount).
  • Фильтры: название/id, статус active/inactive.
  • Сортировка по умолчанию: sort_order, затем name.
  • Колонка «Название» — type: 'link', класс js-district-edit; клик открывает модалку редактирования (AJAX GET /admin/districts/{id}). Паттерн — reusable-ui-partials §3.2.

4) Sync объектов района (location_ids)

  • Сервис: app/Services/LocationDistrictSyncService.php — метод syncLocationsForDistrict(District $district, array $locationIds).
  • Валидация: StoreDistrictRequest, UpdateDistrictRequest (trait DistrictLocationIdsRules) — location_ids nullable array, каждый id — объект текущего партнёра; ошибки под полем «Объекты».
  • Объект может быть перенесён из другого района: при выборе в multiselect прежний district_id у других районов обнуляется автоматически (полная замена привязки).
  • Без locations.view поле location_ids не принимается и не синхронизируется; привязку объектов можно менять на странице /admin/locations (select «Район»).

5) Аудит (my_logs)

Запись — через AuditLogger (audit-my-logs). События: district.created, district.updated, district.deleted (категория district).

  • create — снимок: название, сортировка, активность, объекты.
  • update — diff «старое → новое» по тем же полям; лог только при изменениях; объекты — только если выполнялся sync location_ids.
  • delete — «Район удалён: {name}. ID: {id}.» (level = security); при 422 (есть привязанные объекты) лог не пишется.
  • UI — кнопка «История» (иконка fa-clock-rotate-left) в toolbar → GET /admin/districts/logs-data (route('logs.data.district')), модалка includes.logModal.

6) Удаление

  • Если к району привязан хотя бы один объект (locations.district_id, включая отключённые) — 422: «Нельзя удалить район: к нему привязаны объекты».
  • Без объектов — удаление из БД (hard delete). У заявок school_leads.district_id обнуляется (nullOnDelete).

7) Лендинг и заявки

  • На лендинге родитель выбирает район → объект → услугу (см. school-leads-landing).
  • Объекты без district_id на лендинге не показываются.
  • В school_leads сохраняются и district_id, и location_id (история заявки).
  • В CRM заявок: колонка/фильтр «Район» при districts.view; «Объект» — при locations.view.
