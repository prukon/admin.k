Договоры (клиентские): скачивание в интерфейсе

Папка: /docs/documentation
Правило единое для карточки договора в CRM (/client-contracts/{id}) и для вкладки «Мои документы» в учётной записи пользователя.

Какая кнопка показывается

  • Если статус договора signed (Contract::STATUS_SIGNED) — в интерфейсе предлагается только скачивание подписанного файла. Подпись кнопки/ссылки: «Скачать подписанный».
  • Если статус любой другой — предлагается только скачивание оригинала (неподписанного PDF). Подпись: «Скачать оригинал».

Предзаполнение подписанта (SMS)

На GET /client-contracts/{id} в модалке «Отправка на подпись» ФИО и телефон берутся из карточки ученика: parent_lastname, parent_firstname, parent_middlename, phone (см. admin-users.html §2.1). В ContractsController@show эти поля должны быть в select при загрузке $student. В блоке основной информации карточки договора перед «Ученик» выводится строка «Родитель» (parent_full_name).

Замечания

  • Критерий выбора кнопки — именно поле contracts.status, а не наличие signed_pdf_path.
  • Если статус signed, а файл подписанного PDF в хранилище отсутствует или путь в БД не совпадает с диском, скачивание вернёт ошибку (см. обработку в контроллерах скачивания).
  • Файлы хранятся под корнем диска local: storage/app/ + относительный путь из source_pdf_path / signed_pdf_path.

Создание договора из заявки с сайта

Полный сценарий (лид → ученик → договор) описан в school-leads-widget.html (§5.5–5.7).
  • После привязки ученика к заявке в таблице заявок ссылка «Создать договор» ведёт на GET /client-contracts/create?user_id={id} (право contracts.view).
  • Ученик предвыбирается на форме создания (ContractsController@create, preselectedUser).
  • Списание с баланса партнёра — только при POST /client-contracts (загрузка PDF), не при переходе со страницы заявок.
  • В таблице заявок отображается последний договор ученика по created_at; отдельного поля «договор заявки» в contracts нет.
