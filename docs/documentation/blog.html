Блог: админка, ИИ, публикация в VK

Файл: /docs/documentation/blog.html · в кабинете: /doc
Модуль блога включает публичные страницы на сайте, админку статей и категорий, генерацию контента через OpenAI и автоматическую публикацию анонсов в группу ВКонтакте vk.com/kidscrm. Публикация на сайте и в VK — разные шаги: ИИ создаёт черновик; пост в VK уходит только после явной публикации статьи (и при выполнении условий ниже).

1) Публичная часть (без авторизации)

URLМаршрутОписание
/blogblog.indexСписок опубликованных статей
/blog/category/{slug}blog.categoryСтатьи категории
/blog/{slug}blog.showСтраница статьи (пример: /blog/kak-oformit-dogovor-…)

Статья считается видимой на сайте, если одновременно:

  • is_published = true
  • published_at задан и published_at <= now() (scope BlogPost::published())

Черновики и статьи с датой в будущем по прямой ссылке отдают 404.

2) Админка: доступ и навигация

  • Право: blog.view (описание в permissions: «Страница "Блог"»).
  • Middleware: все маршруты /admin/blog/* в группе can:blog.view.
  • Без права — 403; гость — редирект на вход (как у остальных admin-маршрутов).
  • Раздел в админке: вкладки Статьи, Категории, SEO-настройки (admin.blog._toolbar).

3) Статьи: поля и публикация на сайте

Основная модель: App\Models\BlogPost, таблица blog_posts.

  • title, slug (уникальный, латиница и дефисы), excerpt, content (HTML, Purifier в админке).
  • cover_image_path — обложка в storage/app/public/blog/covers/ (публично: /storage/blog/covers/…).
  • SEO: meta_title, meta_description, canonical_url; на странице статьи — OG-теги (og:image = обложка или дефолт из настроек).
  • is_published, published_at — для черновика дата может проставляться автоматически при сохранении.
  • VK: publish_to_vk (bool, по умолчанию true), vk_message (до 500 символов, опционально).

4) ИИ: генерация статей и изображений

  • Кнопка «+ Статья (ИИ)» на списке статей; для существующей статьи — действия улучшения/SEO в модалке.
  • Настройки промптов, бюджета и моделей — /admin/blog/settings (ключи blog.ai.* в settings).
  • Фоновые задачи: RunBlogAiGenerationJob, RunBlogAiGeneratedImageJob, RunBlogAiImageRegenerationJob.
  • Новая статья от ИИ создаётся как черновик (is_published = false); в VK при этом ничего не уходит.
  • Обложка от ИИ записывается в cover_image_path после генерации изображения — это может запустить публикацию в VK, если статья уже опубликована на сайте и включена галочка VK.

5) Публикация в ВКонтакте

5.1) Условия отправки поста на стену

Пост в VK создаётся один раз на статью (запись в blog_post_social_publications). Повторная публикация на сайте после снятия с публикации не создаёт новый пост в VK.

Задача ставится в очередь, когда выполнены все условия:

  • В .env: VK_ENABLED=true, заданы VK_GROUP_ID и VK_GROUP_TOKEN.
  • В настройках блога (/admin/blog/settings): «Публикация в VK» = включено (blog.vk.enabled в settings).
  • На форме статьи включена галочка «Опубликовать в VK» (publish_to_vk).
  • Статья впервые стала видимой на сайте (is_published + published_at <= now()).
  • Есть cover_image_path — без обложки статус VK: «Ждёт обложку».
  • Ещё нет успешной публикации VK (status != published в blog_post_social_publications).

5.2) Содержимое поста VK

На публичной странице статьи нет кнопки «Поделиться в VK» — автопостинг идёт только на стену группы kidscrm через API.

Текст (приоритеты)

  1. Если на форме статьи заполнено поле «Текст для VK» (vk_message, до 500 символов) — используется оно, ИИ не вызывается.
  2. Если поле пустое и в настройках включено «Текст VK через ИИ» (blog.vk.ai_enabled) — вызывается OpenAI (App\Services\BlogVk\BlogVkAiMessageGenerator): по заголовку, excerpt и началу статьи генерируется уникальный анонс (friendly tone, промпт blog.vk.ai_prompt_template). Результат сохраняется в blog_posts.vk_message (можно отредактировать в админке до повторной отправки).
  3. Если ИИ выключен, не настроен OPENAI_API_KEY, превышен дневной бюджет ИИ блога или запрос к OpenAI завершился ошибкой — используется fallback-шаблон (blog.vk.message_template): переменные {title}, {excerpt}, {url}, {category}.

ИИ для VK использует ту же модель и лимит бюджета, что генерация статей блога (blog.ai.model, blog.ai.daily_budget_usd, BlogAiLimiter). Стоимость одного анонса учитывается в дневном лимите (~$0.03 резерва).

Ссылка

URL статьи с UTM добавляется в текст поста (App\Services\BlogVk\BlogVkUrlBuilder):

  • utm_source=vk (VK_UTM_SOURCE)
  • utm_medium=social (VK_UTM_MEDIUM)
  • utm_campaign=blog (VK_UTM_CAMPAIGN)

Фото (обложка)

  • Если задан VK_USER_TOKEN в .env, обложка (cover_image_path) загружается через photos.getWallUploadServer → upload → photos.saveWallPhoto и прикрепляется к wall.post (App\Services\BlogVk\VkWallPhotoUploader).
  • Без VK_USER_TOKEN или при ошибке загрузки — пост уходит только с текстом и ссылкой (без картинки в VK).
  • VK_GROUP_TOKEN (токен сообщества) достаточен для wall.post, но не для загрузки фото (ошибка API 27).
Тип токена пользователя. Для загрузки фото нужен классический user access token формата vk1.a… с правами photos, wall, groups. Токен из OAuth VK ID (vk2.a…, команда blog:vk-user-token) подходит для авторизации, но часто не даёт вызывать photos.getWallUploadServer (ошибки 15 / 1051). В этом случае посты публикуются без фото — смотрите storage/logs/queue.log (VK: загрузка фото не удалась).

VK_ENABLED=true
VK_GROUP_ID=123456789

VK_API_VERSION=5.199
VK_UTM_SOURCE=vk
VK_UTM_MEDIUM=social
VK_UTM_CAMPAIGN=blog

VK_GROUP_ID — числовой ID сообщества (в API: owner_id = -VK_GROUP_ID). Short name kidscrm в URL группы — не подставлять вместо числового ID.

5.3.1) Где взять VK_USER_TOKEN

Нужен токен пользователя — аккаунт должен быть редактором или администратором группы kidscrm.

Важно: старый кабинет dev.vk.com больше не настраивает Standalone-приложения. Приложение создаётся в VK ID (через VK Бизнес ID).

Шаг A — приложение в VK ID

  1. Откройте кабинет VK ID (вход через VK Бизнес ID).
  2. Создайте приложение → платформа Web.
  3. При создании заполните:
    • Базовый домен: oauth.vk.com
    • Доверенный redirect URL (это и есть бывший «redirect URI»): https://oauth.vk.com/blank.html
  4. Скопируйте ID приложения (client_id).

Шаг B — получить токен на сервере (VK ID + PKCE)

php artisan blog:vk-user-token --client-id=ВАШ_CLIENT_ID

Команда (App\Console\Commands\BlogVkUserTokenCommand) выдаст ссылку для браузера, вы разрешите доступ, вставите полный URL после редиректа — получите строку для VK_USER_TOKEN в .env, затем php artisan config:clear.

Опционально в .env: VK_OAUTH_CLIENT_ID=... — тогда флаг --client-id не нужен.

Старый способ «oauth.vk.com/authorize?...&response_type=token» для новых приложений VK ID не работает — нужен OAuth 2.1 с PKCE (команда выше).

Шаг C — если нужно фото в постах (рекомендуется)

Если после деплоя в queue.log при публикации видите photos.getWallUploadServer: Access denied при токене vk2.a…, получите legacy user token vk1.a… (Standalone-приложение в dev.vk.com, если доступно) с scope wall,photos,groups,offline и scope response_type=token, либо согласуйте расширенные права API с поддержкой VK. Без рабочего user token для photos посты всё равно уходят — текстом и ссылкой.

Токен сообщества (VK_GROUP_TOKEN): Управление группой → Работа с API → Ключи доступа, право запись на стене.

5.5) Очередь, планировщик, код

  • Job: App\Jobs\BlogVk\PublishBlogPostToVkJob — очередь database, имя очереди default, tries = 1, timeout = 180 с (запас на вызов OpenAI для текста VK).
  • Повторный пост в VK для той же статьи блокируется, если статус уже published. Статус failed job не обрабатывает автоматически — только кнопка «Повторить» в списке статей.
  • Координатор: App\Services\BlogVk\BlogVkPublicationCoordinator — когда ставить задачу, дедупликация job в jobs, сброс зависшего publishing (>10 мин), cron для отложенных и «ждёт обложку».
  • Observer: App\Observers\BlogPostObserver (событие saved) — при изменении статьи.
  • Текст: App\Services\BlogVk\BlogVkMessageBuilder (+ BlogVkAiMessageGenerator при пустом vk_message).
  • API VK: App\Services\BlogVk\VkWallClient (wall.post), App\Services\BlogVk\VkWallPhotoUploader.
  • Cron: каждую минуту php artisan blog:process-vk-publications (App\Console\Commands\ProcessBlogVkPublicationsCommand, timezone Europe/Moscow).
  • Инфраструктура: на сервере — schedule:run (cron) и worker с queue:work database --queue=default (см. мониторинг очередей). Виджет очередей группирует задачи VK как «Публикация блога в VK».

Логи (storage/logs/queue.log)

При публикации в VK пишутся строки канала queue, например:

  • PublishBlogPostToVkJob started
  • VK AI message generated — успешная генерация текста
  • VK message: AI skipped, using template fallback — использован шаблон
  • VK photos.getWallUploadServer, VK photo uploaded for wall, has_photo: true
  • VK: загрузка фото не удалась, пост без картинки — fallback без фото
  • PublishBlogPostToVkJob published / PublishBlogPostToVkJob error

7) Настройки в админке (/admin/blog/settings)

Доступ: право blog.view (как и остальные страницы /admin/blog/*). Сохранение — POST admin.blog.settings.update.

  • SEO блога, OG по умолчанию, шаблон title статьи.
  • Блок «ИИ: генерация статей» — модель OpenAI, дневной бюджет, промпт статьи, настройки изображений (общий бюджет для VK-анонсов).
  • Блок «ВКонтакте: публикация статей» — см. таблицу ключей settings ниже.

Индикаторы на странице настроек:

  • VK API настроен — VK_ENABLED, VK_GROUP_ID, VK_GROUP_TOKEN в .env.
  • Фото в VK: VK_USER_TOKEN задан / без VK_USER_TOKEN (только текст + ссылка) — наличие VK_USER_TOKEN в .env (не проверяет работоспособность API photos).
