Админка выплат T‑Bank: список, карточка, API таблицы

Файл: /docs/documentation/tbank-admin-payouts.html · см. также T‑Bank (общий раздел), Очереди и планировщик
Раздел описывает страницу списка выплат и связанные маршруты в CRM. Они отделены от «тяжёлого» админа T‑Bank (manage.payment.method.tbank: sm-register, debug, карточки платежей и т.д.).

1) Доступ

  • Permission: tbank.payouts.manage (middleware can:tbank.payouts.manage).
  • Контекст: как и остальной кабинет — auth, 2fa, SetPartner (current_partner).
  • Без права на все перечисленные ниже маршруты ответ 403.

2) Маршруты

  • GET /admin/tinkoff/payouts — HTML: таблица выплат (DataTables), фильтры, информационные блоки.
  • GET /admin/tinkoff/payouts/data — JSON для DataTables (валидация через TinkoffPayoutsDataTableRequest).
  • GET /admin/tinkoff/payouts/columns-settings — JSON: сохранённые видимые колонки таблицы (per user, ключ tinkoff_payouts_index).
  • POST /admin/tinkoff/payouts/columns-settings — тело JSON {"columns":{"column_key":true|false,...}} → {"success":true}.
  • GET /admin/tinkoff/payouts/{id} — карточка выплаты (id — число).
  • POST /tinkoff/payouts/{deal}/pay-now и POST /tinkoff/payouts/{deal}/delay — ручной запуск и отложенная выплата по deal_id (тот же permission).

3) Изоляция по партнёру

  • Список и DataTables: пользователь без роли superadmin видит только выплаты с tinkoff_payouts.partner_id = current_partner. Параметр partner_id в запросе к /data для такого пользователя не расширяет выборку.
  • Superadmin: видит все выплаты; может передать partner_id в /admin/tinkoff/payouts/data для фильтра по партнёру. На странице списка в фильтрах доступен выбор партнёра.
  • Карточка выплаты: если не superadmin и payout.partner_id не совпадает с current_partner — ответ 404 (не 403).

4) Блоки на странице списка

  • Автовыплаты: по данным payment_systems (name=tbank) — у каких партнёров в settings.auto_payout_enabled включено; для superadmin — сводка по выбранным в фильтре партнёрам, для обычного пользователя — по своему партнёру.
  • Интервал обработки отложенных: из Setting::getTinkoffPayoutScheduledIntervalMinutes() (см. глобальные настройки выплат).
  • Просроченные отложенные выплаты: записи, которые должны подхватываться TinkoffRunScheduledPayoutsJob: when_to_run <= now(), status = INITIATED, tinkoff_payout_payment_id IS NULL, completed_at IS NULL. На странице показываются счётчик и до 25 строк со ссылками в карточку. Для не‑superadmin выборка ограничена текущим партнёром.
Отложенные выплаты попадают в очередь только после срабатывания Laravel Scheduler (php artisan schedule:run по cron, обычно каждую минуту). Если worker работает, а выплаты «висят» в INITIATED с прошедшим when_to_run, проверьте cron и индикатор планировщика на вкладке Настройки → Очереди.
