Администраторы и сотрудники с кастомной ролью

Файл: /docs/documentation/admin-role-staff.html · URL: /docs/documentation/admin-role-staff · обзор вкладок: admin-users-section

1) Назначение и модель

Сотрудник (администратор или пользователь с кастомной ролью партнёра) — обычная запись users с фиксированным role_id. Отдельной таблицы профиля нет (в отличие от trainer_profiles у тренеров).
ФИО, email, телефон, пароль, аватар, активность — в users. При создании и обновении через эти страницы users.team_id сбрасывается в null.

2) Доступ

Все маршруты раздела в routes/web.php обёрнуты в middleware(['can:users.view', 'can:users.role.update']).
Без users.view или без users.role.update — 403.
Вкладка в разделе «Пользователи» видна только при users.role.update (см. admin-users-section).

Контроллер: App\Http\Controllers\Admin\RoleStaffUserController.
Запросы: StoreRoleStaffUserRequest, UpdateRoleStaffUserRequest (authorize: users.role.update).
View: resources/views/admin/role_staff/index.blade.php (общий шаблон для обеих страниц).

3) Администраторы — /admin/administrators

Системная роль roles.name = admin. Вкладка «Администраторы».

3.2) UI и колонки

  • Колонки: аватар, ФИО, email, телефон, активен, действия.
  • Фильтры: имя (ФИО / email / телефон), статус (active / inactive).
  • Создание — модалка: имя*, фамилия*, email, телефон, пароль (опционально), активен.
  • Редактирование — аватар (editAvatar), смена пароля → POST /admin/user/{id}/update-password при users.password.update.
  • Удаление — soft delete пользователя из модалки редактирования.
  • Ошибки валидации — JSON 422, поля под формой.

4) Кастомные роли — /admin/roles/{name}

{name} — машинное имя роли (roles.name), только [a-z0-9_]+. Роль должна быть несистемной (is_sistem = 0) и привязана к текущему партнёру через partner_role. Подпись вкладки — roles.label из настроек роли.

6) Изоляция по партнёру

  • Список и операции — только users.partner_id = current_partner и ожидаемый role_id.
  • Чужой партнёр или пользователь другой роли на role-scoped маршруте → 404.
  • Кастомная роль другого партнёра → 404 на /admin/roles/{name}.
