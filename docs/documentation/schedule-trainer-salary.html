ЗП тренеров

Файл: /docs/documentation/schedule-trainer-salary.html · страницы: GET /schedule/trainer-salary (schedule.trainer-salary), GET /schedule/trainer-salary/data (schedule.trainer-salary.data)
Не путать с «Расписание школы» (слоты абонементов).
Расчёт строится из журнала посещаемости (schedule_users, статус «Посетил», trainer_profile_id) — так же, как «Нагрузка тренеров».
Вкладка «ЗП тренеров» в разделе «Журнал расписания» (оболочка resources/views/admin/schedule/index.blade.php).
За календарный месяц ведётся черновик в БД (autosave). Кнопки «Сформировать» / «Сформировать всех» создают версионные слепки — их архив на вкладке «Листы ЗП».

3) Справочник: оклад и ставка по умолчанию

В карточке тренера (/admin/trainers) поля default_base_salary, default_rate_per_training (trainer_profiles, миграция 2026_05_22_100000_add_salary_defaults_to_trainer_profiles_table.php).

При первом появлении тренера в черновике за месяц значения копируются в строку trainer_salary_draft_lines. Дальнейшее изменение в черновике не меняет справочник в карточке тренера.

5) Кто попадает в таблицу

Активные тренеры партнёра (TrainerSalaryService::activeTrainersForPartner):

  • trainer_profiles.is_enabled = true;
  • связанный users.is_enabled = true;
  • сортировка: sort_order, затем id.

Отключённый тренер или пользователь в таблице ЗП не показывается; существующая строка черновика при отключении не удаляется автоматически.

6) Кол-во тренировок

Совпадает с итогом по строке тренера в «Нагрузке тренеров» при режиме «без групп» (суммы по дням недели): один календарный день может учитываться дважды, если в этот день были визиты учеников из разных групп с одним тренером.

Сервис: TrainerWorkloadReportService::trainerRowTrainingsTotals($partnerId, $dateFrom, $dateTo) — вызывается при buildReport и перед формированием слепка.

7) Формула итого

итого = оклад + (кол-во_тренировок × ставка) + бонусы − вычеты

Расчёт только на сервере: App\Services\Schedule\TrainerSalaryCalculator (денежные поля — 2 знака, bc*).

Пример: оклад 10000, 3 тренировки × 300, бонусы 0, вычеты 0 → сумма за тренировки 900, итого 10900.

8) UI и фронтенд

  • Шаблон: resources/views/admin/schedule/trainer_salary.blade.php, фрагмент таблицы: _trainer_salary_table.blade.php.
  • Скрипт: resources/js/trainer-salary.js (Vite на вкладке).
  • Фильтр «Период» — input type="month"; смена месяца → GET …/data, обновление URL query (year, month).
  • При manage: autosave черновика по change/blur полей (debounce 400 мс) → PATCH …/draft/{id}; ошибки 422 — под соответствующим полем (data-error-for).
  • Кнопка «Сформировать» в строке → POST …/snapshots/{id}; под именем тренера — подсказка «Слепок vN · дата · кто сформировал».
  • «Сформировать всех» — confirm, затем POST …/snapshots; создаёт слепок v1 для каждого активного тренера с одним batch_id.
  • Без manage: денежные поля и комментарий readonly, колонки «Действие» нет.

9) Бэкенд

  • Контроллер: ScheduleTrainerSalaryController.
  • Сервис: TrainerSalaryService — период, синхронизация строк, пересчёт тренировок, PATCH, слепки.
  • Запросы: TrainerSalaryReportRequest (view), UpdateTrainerSalaryDraftLineRequest, FormTrainerSalarySnapshotRequest (manage; authorize + rules + messages).

9.1) Ответ GET …/data

JSON: year, month, month_label, date_from, date_to, can_manage, table_html, rows[].

Элемент rows[]: trainer_profile_id, trainer_name, денежные поля строкой "NNNN.NN", trainings_count, comment, latest_snapshot (или null: id, version, batch_id, formed_at, formed_by_name, total).

10) Версии слепков

  • «Сформировать» по тренеру: version = max(version) + 1 в рамках периода и тренера; batch_id = null.
  • «Сформировать всех»: для каждого активного тренера слепок v1 (или следующая версия, если уже были одиночные слепки), общий batch_id (UUID).
  • Слепок фиксирует все поля строки на момент формирования; черновик после этого можно менять — новый слепок возьмёт актуальный черновик.
