Настройки → Логи: журнал изменений (my_logs)

Файл: /docs/documentation/settings-logs.html
Назначение. Отдельная вкладка «Настройки → Логи» (/admin/settings/logs) — единый журнал аудита по таблице my_logs для текущего партнёра. Раньше тот же контент открывался модальным окном «История изменений» на вкладке «Общие»; модалка убрана, просмотр только через вкладку.
Permission viewing.all.logs (в справочнике: «Просмотр всех логов») означает доступ к журналу всех типов событий в рамках разрешённого partner-scope, а не «логи всех партнёров» для обычного админа. Cross-partner (все школы сразу) — только у роли superadmin на этой вкладке. На отчёте «Платежи» то же право даёт кнопку «История» T‑Bank — см. отчёт «Платежи».

1) Доступ и права

ЧтоPermission / рольПримечание

Контекст CRM: auth + 2fa + SetPartner. См. также партнёр-контекст и права в разрезе партнёра.

2) Маршруты

  • GET /admin/settings/logs — HTML (вкладка внутри admin.setting.index, activeTab = logs).
  • GET /admin/settings/logs-data — JSON для jQuery DataTables (server-side).

Имена маршрутов: admin.setting.logs, settings.logs.data (routes/web.php).

3) Интерфейс

  • Шапка в стиле отчёта «Платежи» / «Комиссии Т‑Банк»: карточка payments-report-*, CSS resources/css/admin-list-toolbar.css (подключается через Vite на вкладке).
  • Кнопка «Фильтры» — сворачиваемая панель с полями фильтрации и кнопками «Применить» / «Сброс» (перезагрузка DataTables без перезагрузки страницы).
  • Таблица — server-side DataTables, по умолчанию 10 строк на странице, сортировка по умолчанию: «Дата создания» по убыванию.

3.1 Колонки таблицы

  1. ID
  2. Дата создания (created_at, формат d.m.Y / H:i:s)
  3. Партнёр — только для superadmin (колонка partner_title)
  4. Действие (человекочитаемая подпись из MyLog::actionLabels())
  5. Автор (author → ФИО пользователя)
  6. Что меняли (target_label)
  7. Описание (description, переносы строк → <br> в ячейке)

4) Partner-scope: режим SUPERADMIN_ALL_OR_FILTER

На этой вкладке для API логов включён режим App\Support\PartnerScopeMode::SUPERADMIN_ALL_OR_FILTER (реализация в PartnerContext::applyPartnerScope(), вызов из трейта BuildsLogTable только из SettingController::logsData).
  • STRICT_CURRENT (по умолчанию в остальных местах) — всегда where partner_id = current_partner, если партнёр в контексте задан.
  • SUPERADMIN_ALL_OR_FILTER (только вкладка «Логи»):
    • superadmin + filter_partner_id пусто / all → без фильтра по partner_id (все партнёры);
    • superadmin + числовой filter_partner_id → один партнёр;
    • не superadmin → как STRICT_CURRENT (подмена all в AJAX игнорируется на сервере).

Подробнее о контексте и anti-leak: partner-context.html.

5) Данные и код

  • Таблица БД: my_logs (partner_id, type, action, author_id, target_*, description, created_at).
  • Билдер: трейт App\Support\BuildsLogTable — единый DataTables для логов на разных страницах (пользователи, группы, расписание, партнёры, настройки цен, правила…). На вкладке «Логи» передаётся type = null (все типы).
  • Подписи действий: MyLog::actionLabels() — общий словарь кодов action.
  • Контроллер: App\Http\Controllers\Admin\Setting\SettingController — showLogs(), logsData().
  • View: resources/views/admin/setting/logs.blade.php (подключается из admin/setting/index.blade.php).

5.1 Другие экраны с логами (без изменений)

Модалки «История изменений» на других страницах по-прежнему используют BuildsLogTable с режимом по умолчанию (STRICT_CURRENT) и фильтром по type:

  • пользователи — GET /admin/user/logs-data (type = 2);
  • группы — GET /admin/teams/logs-data (type = 3);
  • расписание — GET /schedule/logs-data (type = 9);
  • права и роли — GET /admin/setting/rules/logs-data;
  • партнёры — GET /admin/partner/logs-data (type = 80);
  • цены — GET /admin/setting-prices/logs-data (type = 1).

Не путать с вкладкой «Настройки → Очереди» и файлом storage/logs/queue.log — это операционный лог воркера, не my_logs (очереди в админке).

6) Отличие от вкладки «Общие»

  • На вкладке «Общие» (/admin/settings) кнопка «История изменений» и модалка #historyModal удалены.
  • Журнал всех типов событий партнёра перенесён на вкладку «Логи» с расширенными фильтрами и (для superadmin) выбором партнёра.
