Виды спорта (админка): /admin/sport-types

Файл: /docs/documentation/admin-sport-types.html · страница CRM: GET /admin/sport-types · связь с группами: /admin/teams · лендинг заявки: school-leads-landing §4.4
Справочник «Вид спорта» на уровне партнёра. Один вид спорта может быть привязан к нескольким группам (teams); у группы поле sport_type_id необязательно (nullable).
Пункт меню «Виды спорта» в сайдбаре — при праве sport_types.view (@can в sidebar.blade.php, Gate в AuthServiceProvider).
Все маршруты раздела в routes/web.php обёрнуты в middleware('can:sport_types.view'). Создание, изменение и удаление — дополнительно sport_types.manage (middleware на POST/PUT/DELETE и authorize() в FormRequest).

1) Модель данных

Миграции:

  • 2026_05_29_120000_create_sport_types_table.php — таблица sport_types;
  • 2026_05_29_120001_add_sport_type_id_to_teams_table.php — teams.sport_type_id, FK nullOnDelete;
  • 2026_05_29_120002_add_sport_type_id_to_school_leads_table.php — school_leads.sport_type_id, FK nullOnDelete;
  • 2026_05_29_120003_add_sport_types_permissions.php — права sport_types.view, sport_types.manage.
Таблица / полеНазначение
sport_types partner_id, name (уникально в рамках партнёра), description, sort, is_enabled
teams.sport_type_id Необязательная привязка группы к виду спорта. При удалении вида спорта — SET NULL (не блокирует удаление)
school_leads.sport_type_id Выбор родителя на лендинге заявки. В CRM-таблице заявок колонка пока не выведена (поле есть в БД)

2) Права доступа

PermissionОписаниеЧто открывает
sport_types.view Страница «Виды спорта» Меню, index, data, show, columns-settings; на /admin/teams — колонка, фильтр, селект в модалках (см. admin-teams)
sport_types.manage Виды спорта: создание/редактирование POST store, PUT update, DELETE destroy; кнопка «Добавить» и модалки create/edit на странице справочника

Без sport_types.view — 403 на все эндпоинты раздела; для гостя — редирект/401. При только sport_types.view страница и чтение доступны, мутации возвращают 403.

Права sport_types.view и sport_types.manage не входят в базовый набор для новых партнёров (config/role_base_permissions.php, роль admin) — включаются вручную в матрице прав. Миграция 2026_05_29_120003 только добавляет записи в permissions, без автоназначения permission_role. Общая модель: Партнёры: базовые роли и права.

4) Интерфейс страницы

View: resources/views/admin/sport-types/index.blade.php. Общий toolbar — переиспользуемые UI-блоки, CSS admin-list-toolbar.css.

  • Toolbar — «Добавить» (sport_types.manage), «Фильтры», «Колонки».
  • Фильтры (collapse): name (название/описание/id), status — active/inactive; плюс глобальный поиск DataTables.
  • DataTables — server-side, ключ настроек колонок sport_types_index.
  • Колонки (настраиваемые): sort, name, teams_count (число связанных групп), is_enabled_label, actions (при sport_types.manage).
  • Модалки create/edit — поля: name, description, sort, is_enabled. Ошибки валидации — JSON 422, тексты на русском под полями.
  • Уникальность — (partner_id, name); дубликат → 422 «Вид спорта с таким названием уже существует».

5) Связь с группами (teams)

Привязка выполняется на странице /admin/teams, а не в справочнике видов спорта:

  • В модалках create/edit группы — селект sport_type_id (только активные виды спорта партнёра), если у пользователя есть sport_types.view и в справочнике есть записи.
  • В таблице групп — колонка «Вид спорта» (sport_type_label) и фильтр #filter-sport-type (id или none — группы без вида спорта).
  • Без sport_types.view ключ sport_type_id отбрасывается в TeamController до сохранения; UI скрыт.
  • Валидация: App\Rules\PartnerSportTypeId — id должен принадлежать текущему партнёру и иметь is_enabled = true. Дополнительная проверка в TeamService::resolveSportTypeIdForPartner().
Вид спорта группы и локация группы — независимые измерения. Локации по-прежнему управляются через location_team (Локации ↔ группы). Вид спорта не заменяет район/локацию на лендинге.

6) Лендинг заявки (/lead/{landingKey})

Если у партнёра есть хотя бы один активный вид спорта, на форме появляется необязательный селект «Вид спорта». Подробно: school-leads-landing §4.4.

  • GET /lead/{landingKey}/teams?location_id=&sport_type_id= — фильтрует услуги (группы) по району и опционально по виду спорта.
  • POST /lead/{landingKey}/submit — сохраняет sport_type_id в school_leads; валидация отклоняет чужой/disabled вид спорта и несовпадение team_id с выбранным видом спорта.
  • Если справочник видов спорта пуст — блок на форме не рендерится, поведение как раньше (только район и услуга).
