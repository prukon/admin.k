Отчёты (админка): фильтры/AJAX/колонки/суммы

Файл: /docs/documentation/reports-admin.html · список разделов (/doc)

1) Общие принципы UI/UX

Начиная с 2026‑03, отчёты админки приводятся к единому каркасу (как в “Платежи”):
  • верхняя панель: заголовок + “Общая сумма” + кнопки “Фильтры”/“Колонки”
  • применение фильтров без перезагрузки страницы: preventDefault() → table.ajax.reload()
  • “Общая сумма” обновляется через отдельный endpoint …/total и меняется с анимацией
  • настройки колонок сохраняются для текущего пользователя в user_table_settings (если отчёт поддерживает “Колонки”)

2) Права доступа (middleware can:…)

Важно: разные отчёты находятся в разных группах прав. Без соответствующего permission ответ будет 403 (страница и все AJAX endpoints).
  • reports.view — отчёты в группе /admin/reports/* (Платежи, Задолженности, LTV, Платежи по месяцам).
  • reports.payment.intents.view — отчёт “Платежные запросы”.
  • reports.fiscal.receipts.view — отчёт “Чеки”.
Модель прав в разрезе партнёра: partners-permissions.

3) Отчёт “Задолженности”

  • Страница: /admin/reports/debts (middleware can:reports.view)
  • DataTables: GET /admin/reports/getDebts (AJAX)
  • Общая сумма: GET /admin/reports/debts/total (JSON: { total_formatted, total_raw })

4) Отчёт “Платежи по ученикам (LTV)”

  • Страница: /admin/reports/ltv (middleware can:reports.view)
  • DataTables: GET /admin/reports/ltv/data (AJAX)
  • Детализация: GET /admin/reports/ltv/{user}/payments (AJAX)
  • Общая сумма: GET /admin/reports/ltv/total (JSON: { total_formatted, total_raw })
“Общая сумма” в LTV — это sum(payments.summ) по тем же фильтрам (и по текущему партнёру), что и таблица.

5) Отчёт “Платежи по месяцам”

  • Страница: /admin/reports/payments/monthly (middleware can:reports.view)
  • DataTables (месяцы): GET /admin/reports/payments/monthly/data (AJAX)
  • Детализация месяца: GET /admin/reports/payments/monthly/{yearMonth}/payments (AJAX)
  • Общая сумма: GET /admin/reports/payments/monthly/total (JSON: { total_formatted, total_raw })
Режим группировки (по дате операции / по месяцу абонемента) в UI хранится в форме/JS и не обязан отражаться в URL. В запросы DataTables/детализации/total передаётся параметр mode.

6) Отчёт “Платежные запросы” (payment_intents)

  • Страница: /admin/reports/payment-intents (middleware can:reports.payment.intents.view)
  • DataTables: GET /admin/reports/getPaymentIntents (AJAX)
  • Общая сумма: GET /admin/reports/payment-intents/total (JSON: { total_formatted, total_raw })
  • Колонки (настройки):
    • GET /admin/reports/payment-intents/columns-settings
    • POST /admin/reports/payment-intents/columns-settings (columns: {key: true/false})
    • Хранение: user_table_settings.table_key = reports_payment_intents
  • Select2:
    • GET /admin/reports/payment-intents/partners-search → { results: [{id,text}] }
    • GET /admin/reports/payment-intents/users-search → { results: [{id,text}] } (поддерживает partner_id)

7) Отчёт “Чеки” (fiscal_receipts)

  • Страница: /admin/reports/fiscal-receipts (middleware can:reports.fiscal.receipts.view)
  • DataTables: GET /admin/reports/fiscal-receipts/data (AJAX)
  • Общая сумма: GET /admin/reports/fiscal-receipts/total (JSON: { total_formatted, total_raw }, считает sum(fiscal_receipts.amount))
  • Колонки (настройки):
    • GET /admin/reports/fiscal-receipts/columns-settings
    • POST /admin/reports/fiscal-receipts/columns-settings (columns: {key: true/false})
    • Хранение: user_table_settings.table_key = reports_fiscal_receipts
  • Select2: GET /admin/reports/fiscal-receipts/partners-search → { results: [{id,text}] }
